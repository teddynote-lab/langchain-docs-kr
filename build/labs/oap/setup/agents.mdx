Open Agent Platform 설정의 첫 번째 단계는 agent를 배포하고 구성하는 것입니다. 이를 돕기 위해 Open Agent Platform에 특별히 맞춤화된 세 가지 사전 구축된 agent를 제공합니다:

- [Tools Agent](https://github.com/langchain-ai/oap-langgraph-tools-agent)
- [Supervisor Agent](https://github.com/langchain-ai/oap-agent-supervisor)
- [Deep Research Agent](https://github.com/langchain-ai/oap-deep-researcher)

## Agent 배포하기

인스턴스에서 이러한 agent를 사용하려면 다음을 수행해야 합니다:

1. repository를 clone합니다
2. README의 지침을 따릅니다
3. agent를 LangSmith에 배포하거나, `langgraph dev`를 실행하여 로컬에서 agent를 실행합니다.

## API Key

이러한 agent는 model을 위한 API key가 필요하며, 경우에 따라 다른 기능(예: 검색 도구)을 위한 API key도 필요합니다.
개발자로서 사용자가 Open Agent Platform의 설정 페이지에서 이러한 API key를 구성할 수 있도록 허용할 수 있습니다.
데모 agent에서 사용할 수 있도록 `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `GOOGLE_API_KEY`, `TAVILY_API_KEY`를 노출했지만, 사용자 정의 agent를 위해 쉽게 확장할 수 있습니다.


사용자가 이 페이지에서 설정한 key는 runtime config의 일부로 `apiKeys` configurable field 아래에서 모든 agent에 전달됩니다.
개발자로서 사용자가 자신의 API key를 가져오도록 요구하거나, LangSmith 배포 자체에 설정된 environment variable로 fallback하도록 선택할 수 있습니다.

    <Tip>
    OAP의 데모 인스턴스를 사용할 때, Tools Agent와 Supervisor는 먼저 사용자가 OAP에서 설정한 API key를 사용하지만, 기본 배포에서 LangChain 팀이 설정한 API Key로 fallback합니다.
    Deep Researcher는 데모 인스턴스에서 사용자가 OAP에서 자신의 API key를 설정해야 **합니다**.
    </Tip>

## 구성

배포되거나 로컬에서 실행되면, configuration environment variable을 설정하여 Open Agent Platform 인스턴스에 연결할 수 있습니다. configuration 객체는 다음과 같습니다:

```json
{
  "id": "The project ID of the deployment. For locally running LangGraph servers, this can be any UUID Version 4.",
  "tenantId": "The tenant ID of your LangSmith account. For locally running LangGraph servers, this can be any UUID Version 4.",
  "deploymentUrl": "The API URL to your deployment.",
  "name": "A custom name for your deployment",
  "isDefault": "Whether this deployment is the default deployment. Should only be set to true for one deployment.",
  "defaultGraphId": "The graph ID of the default graph for the entire OAP instance. We recommend this is set to the graph ID of a graph which supports RAG & MCP tools. This must be set in the same deployment which isDefault is set to true on. Optional, but required in at least one deployment.",
}
```
    <Tip>
    `langgraph dev`를 통해 로컬에서 agent를 실행하는 경우, `id` (project ID)와 `tenantId`는 `uuid.uuid4()`로 생성된 것과 같은 유효한 UUID version 4여야 합니다. 각 graph는 고유한 `id`를 가지고 있어야 하며, 모든 graph는 동일한 `tenantId`를 공유해야 합니다.
    </Tip>

## Project & Tenant ID 찾기

LangSmith에 배포된 agent의 project & tenant ID를 쉽게 찾으려면, 배포 URL의 `/info` endpoint에 GET 요청을 할 수 있습니다. 그런 다음 `project_id` 값을 `id`에 복사하고, `tenant_id`를 `tenantId`에 복사합니다.

## Environment Variable 설정

Open Agent Platform 인스턴스에 포함하려는 각 배포에 대해 이러한 값으로 JSON 객체를 구성한 후, 단일 flat array로 stringify한 다음 `apps/web/` 디렉토리 내의 `NEXT_PUBLIC_DEPLOYMENTS` environment variable 아래에 설정해야 합니다.

다음은 단일 로컬 배포에 대한 이 variable의 예시입니다:

```bash
NEXT_PUBLIC_DEPLOYMENTS=[{"id":"bf63dc89-1de7-4a65-8336-af9ecda479d6","deploymentUrl":"http://localhost:2024","tenantId":"42d732b3-1324-4226-9fe9-513044dceb58","name":"Local deployment","isDefault":true,"defaultGraphId":"agent"}]
```
