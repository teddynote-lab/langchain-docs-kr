<Info>
**사전 요구사항:**
- [Corepack](https://github.com/nodejs/corepack?tab=readme-ov-file#how-to-install) - 플랫폼을 로컬에서 빌드하고 실행하기 위해 필요
- [LangSmith](https://smith.langchain.com/) 계정 (무료 티어로 충분)
- [Supabase](https://supabase.com/) 계정
- MCP Server (예: [Arcade](https://arcade-ai.com/))
- LLM API Key (예: [OpenAI](https://platform.openai.com/), [Anthropic](https://console.anthropic.com/), [Google](https://aistudio.google.com/))
</Info>

## 1. Open Agent Platform 코드 다운로드

<Steps>
    <Step title="Repository 복제">
    GitHub에서 [Open Agent Platform Repository](https://github.com/langchain-ai/open-agent-platform)를 복제합니다
    </Step>
    <Step title="환경 변수 설정">
        <Tip>
        Repository에는 플랫폼 실행에 필요한 모든 환경 변수가 나열된 `.env.example` 파일이 포함되어 있습니다.
        </Tip>
    다음 환경 변수를 설정합니다:
    ```bash
      NEXT_PUBLIC_BASE_API_URL="http://localhost:3000/api"
      LANGSMITH_API_KEY="lsv2_..."
      # Or whichever LLM's API key you're using
      OPENAI_API_KEY="..."
    ```
    </Step>
</Steps>

## 2. 인증 설정

Open Agent Platform은 기본적으로 Supabase를 인증에 사용합니다.

<Steps>
    <Step title="Supabase 프로젝트 생성">
    [Supabase](https://supabase.com/)에서 새 프로젝트를 생성합니다.
    </Step>
    <Step title="환경 변수 구성">
    `apps/web/` 디렉토리 내에 다음 환경 변수를 설정합니다:
        <Tip>
        이러한 Next.js 전용 변수는 Supabase 대시보드의 "Connect" 창에서 "App Frameworks" 아래에서 찾을 수 있습니다
        </Tip>
    ```bash
    NEXT_PUBLIC_SUPABASE_URL="<your supabase url>"
    NEXT_PUBLIC_SUPABASE_ANON_KEY="<your supabase anon key>"
    ```
    </Step>
    <Step title="인증 제공자 활성화">
        Supabase 프로젝트에서 Google 인증을 활성화하거나, UI에서 Google을 인증 옵션으로 표시하지 않으려면 환경 변수에 `NEXT_PUBLIC_GOOGLE_AUTH_DISABLED=true`를 설정합니다.
    </Step>
</Steps>

## 3. Agent 배포

Open Agent Platform 설정의 다음 단계는 agent를 배포하고 구성하는 것입니다.

<Steps>
    <Step title="사전 구축된 Agent 복제">
    Open Agent Platform을 위해 특별히 제작된 세 가지 사전 구축 agent를 출시했습니다:
    - [Tools Agent](https://github.com/langchain-ai/oap-langgraph-tools-agent)
    - [Supervisor Agent](https://github.com/langchain-ai/oap-agent-supervisor)
    - [Deep Research Agent](https://github.com/langchain-ai/oap-deep-researcher)
    </Step>
    <Step title="Agent 설정 지침 따르기">
    각 agent repository에 대해:
    1. Repository를 복제합니다
    2. README의 지침을 따릅니다
    3. Agent를 LangSmith에 배포하거나, `langgraph dev`를 실행하여 agent를 로컬에서 실행합니다. 선택적으로 `--port <port>`를 전달하여 사용자 정의 포트를 사용할 수 있습니다. 이는 여러 graph를 로컬에서 실행할 때 유용합니다.
    </Step>
    <Step title="환경 변수 구성">
    배포 후, 각 agent에 대한 구성 객체를 생성합니다 (다음 단계를 위해):
    ```json
    {
      "id": "The project ID of the deployment",
      "tenantId": "The tenant ID of your LangSmith account",
      "deploymentUrl": "The API URL to your deployment",
      "name": "A custom name for your deployment",
      "isDefault": "Whether this deployment is the default deployment (only one can be default)",
      "defaultGraphId": "The graph ID of the default graph (optional, only required if isDefault is true)"
    }
    ```

    LangSmith 배포의 `/info` endpoint에 GET 요청을 보내면 project 및 tenant ID를 찾을 수 있습니다.

        <Tip>
        `langgraph dev`를 통해 agent를 로컬에서 실행하는 경우, `id` (project ID)와 `tenantId`는 `uuid.uuid4()`로 생성된 것과 같은 유효한 UUID 버전 4여야 합니다. 각 graph는 고유한 `id`를 가져야 하며, 모든 graph는 동일한 `tenantId`를 공유해야 합니다.
        </Tip>
    </Step>
    <Step title="환경 변수 설정">
    Agent 구성을 JSON 배열로 결합하고 `apps/web/` 디렉토리 내에 `NEXT_PUBLIC_DEPLOYMENTS` 환경 변수를 설정합니다:

    ```bash
    NEXT_PUBLIC_DEPLOYMENTS=[{"id":"bf63dc89-1de7-4a65-8336-af9ecda479d6","deploymentUrl":"http://localhost:2024","tenantId":"42d732b3-1324-4226-9fe9-513044dceb58","name":"Local deployment","isDefault":true,"defaultGraphId":"agent"}]
    ```
    </Step>
</Steps>

## 4. RAG Server 설정

<Steps>
    <Step title="LangConnect 배포">
    [LangConnect README](https://github.com/langchain-ai/langconnect)의 지침에 따라 LangConnect RAG server를 설정하고 배포합니다.
    </Step>
    <Step title="환경 변수 구성">
    `apps/web/` 디렉토리 내에 RAG server URL을 설정합니다:
    ```bash
    NEXT_PUBLIC_RAG_API_URL="http://localhost:8080"
    ```
    </Step>
</Steps>

## 5. MCP Server 설정

<Tip>
Open Agent Platform은 Streamable HTTP 요청을 지원하는 MCP server에만 연결할 수 있습니다.
아직 MCP server가 설정되지 않은 경우, [Arcade](https://docs.arcade.dev/home/mcp-desktop-clients/vscode-client)의 공개 베타 MCP server를 사용할 수 있습니다.
</Tip>

<Steps>
    <Step title="MCP Server URL 구성">
    `apps/web/` 디렉토리 내에 MCP server URL을 설정합니다 (`/mcp`로 끝나지 않도록 하세요. 이는 OAP에 의해 자동으로 적용됩니다):
    ```bash
    NEXT_PUBLIC_MCP_SERVER_URL="https://api.arcade.dev/v1/mcps/arcade-anon/"
    ```
    </Step>
    <Step title="인증 구성 (필요한 경우)">
    인증이 필요한 MCP server의 경우:
    ```bash
    NEXT_PUBLIC_MCP_AUTH_REQUIRED=true
    ```
    </Step>
</Steps>

## 6. 플랫폼 실행

구성된 환경 변수로 애플리케이션을 시작합니다:

```bash
# Navigate to the web app directory
cd apps/web

# Install dependencies
yarn install

# Start the development server
yarn dev
```

이제 Open Agent Platform이 http://localhost:3000에서 실행됩니다!
