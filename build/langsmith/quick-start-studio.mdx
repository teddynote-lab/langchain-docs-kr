[LangSmith Deployments UI](https://smith.langchain.com)의 [Studio](/langsmith/studio)는 두 가지 유형의 graph 연결을 지원합니다:

- [클라우드 또는 자체 호스팅](#deployed-graphs)에 배포된 graph.
- [LangGraph server](#local-development-server)로 로컬에서 실행되는 graph.

## 배포된 graph

Studio는 **Deployments** 네비게이션에서 [LangSmith UI](https://smith.langchain.com)를 통해 접근할 수 있습니다.

[배포된](/langsmith/deployment-quickstart) 애플리케이션의 경우, 해당 배포의 일부로 Studio에 접근할 수 있습니다. 이를 위해 UI에서 배포로 이동하여 **Studio**를 선택하세요.

이렇게 하면 라이브 배포에 연결된 Studio가 로드되어, 해당 배포의 [thread](/oss/python/langgraph/persistence#threads), [assistant](/langsmith/assistants), [memory](/oss/python/concepts/memory)를 생성, 읽기, 업데이트할 수 있습니다.

## 로컬 개발 서버

### 사전 요구사항

Studio를 사용하여 애플리케이션을 로컬에서 테스트하려면:

- 먼저 [로컬 애플리케이션 빠른 시작](/langsmith/local-server)을 따르세요.
- 데이터가 LangSmith로 [추적](/langsmith/observability-concepts#traces)되지 않도록 하려면, 애플리케이션의 `.env` 파일에 `LANGSMITH_TRACING=false`를 설정하세요. 추적이 비활성화되면 로컬 서버에서 데이터가 외부로 전송되지 않습니다.

### 설정

1. [LangGraph CLI](/langsmith/cli)를 설치하세요:

    <CodeGroup>
    ```bash pip
    pip install -U "langgraph-cli[inmem]"
    langgraph dev
    ```

    ```bash uv
    uv add langgraph-cli[inmem]
    langgraph dev
    ```

    ```bash npm
    npx @langchain/langgraph-cli dev
    ```
    </CodeGroup>

    <Warning>
    **브라우저 호환성**
    Safari는 Studio로의 `localhost` 연결을 차단합니다. 이 문제를 해결하려면 `--tunnel` 옵션과 함께 명령을 실행하여 보안 터널을 통해 Studio에 접근하세요.
    </Warning>

    이렇게 하면 LangGraph Server가 로컬에서 인메모리로 실행됩니다. 서버는 watch 모드로 실행되어 코드 변경 사항을 감지하고 자동으로 재시작합니다. API 서버를 시작하는 모든 옵션에 대해 알아보려면 이 [레퍼런스](/langsmith/cli#dev)를 읽어보세요.

    다음과 같은 로그가 표시됩니다:

    ```
    > Ready!
    >
    > - API: [http://localhost:2024](http://localhost:2024/)
    >
    > - Docs: http://localhost:2024/docs
    >
    > - LangSmith Studio Web UI: https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024
    ```

    실행되면 자동으로 Studio로 이동됩니다.

1. 실행 중인 서버의 경우, 다음 중 하나로 Dbugger에 접근하세요:
    1. 다음 URL로 직접 이동: `https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024`.
    1. UI에서 **Deployments**로 이동하여 배포의 **Studio** 버튼을 클릭하고, `http://127.0.0.1:2024`를 입력한 후 **Connect**를 클릭하세요.

    다른 호스트나 포트에서 서버를 실행하는 경우, `baseUrl`을 일치하도록 업데이트하세요.

### (선택사항) debugger 연결

중단점과 변수 검사를 통한 단계별 디버깅을 위해 다음을 실행하세요:

    <CodeGroup>
    ```bash pip
    # Install debugpy package
    pip install debugpy
    # Start server with debugging enabled
    langgraph dev --debug-port 5678
    ```

    ```bash uv
    # Install debugpy package
    uv add debugpy
    # Start server with debugging enabled
    langgraph dev --debug-port 5678
    ```
    </CodeGroup>

그런 다음 선호하는 debugger를 연결하세요:

<Tabs>
    <Tab title="VS Code">
    `launch.json`에 다음 구성을 추가하세요:

    ```json
    {
        "name": "Attach to LangGraph",
        "type": "debugpy",
        "request": "attach",
        "connect": {
          "host": "0.0.0.0",
          "port": 5678
        }
    }
    ```
    </Tab>
    <Tab title="PyCharm">
    1. Run → Edit Configurations로 이동
    2. +를 클릭하고 "Python Debug Server"를 선택
    3. IDE host name 설정: `localhost`
    4. port 설정: `5678` (또는 이전 단계에서 선택한 포트 번호)
    5. "OK"를 클릭하고 디버깅 시작
    </Tab>
</Tabs>

<Tip>
시작하는 데 문제가 있는 경우 [문제 해결 가이드](/langsmith/troubleshooting-studio)를 참조하세요.
</Tip>
