## Overview

이 가이드는 PagerDuty의 [Events API v2](https://developer.pagerduty.com/docs/events-api-v2-overview)를 사용하여 [LangSmith 알림](/langsmith/alerts)의 알림 채널로 PagerDuty를 구성하는 과정을 안내합니다. 이 통합을 통해 중요한 LLM 애플리케이션 문제가 PagerDuty incident를 트리거하여 기존 incident 관리 워크플로우를 통해 신속하게 대응할 수 있습니다.

## Prerequisites

* 관리자 액세스 권한이 있는 활성 PagerDuty 계정
* PagerDuty의 적절한 서비스 수준 권한

또한 LangSmith의 커스텀 배포를 사용하는 경우, LangSmith 서비스에서 나가는 트래픽을 차단하는 방화벽 설정이 없는지 확인하세요.

## Integration Steps

### Step 1: Create a Service in PagerDuty

1. PagerDuty 계정에 로그인합니다

2. **Services → Service Directory**로 이동합니다

3. **+ New Service**를 클릭합니다

4. 다음 필드를 작성합니다:

   * **Name**: 설명이 포함된 이름을 입력합니다 (예: "LangSmith Monitoring")
   * **Description**: 모니터링되는 애플리케이션에 대한 세부 정보를 추가합니다
   * **Escalation Policy**: 적절한 팀 escalation policy를 선택합니다
   * **Integration Type**: "Events API V2"를 선택합니다

5. **Add Service**를 클릭하여 서비스를 생성합니다

### Step 2: Obtain Integration Key

서비스를 생성한 후 Integration Key를 가져와야 합니다:

1. Service 드롭다운 아래의 **Service Directory**에서 새로 생성한 서비스를 찾아 클릭합니다
2. **Integrations** 탭을 선택합니다
3. "Events API V2" integration을 찾습니다
4. **Integration Key**(32자 영숫자 문자열)를 복사합니다

![PagerDuty Integration Key Location](/langsmith/images/pager-duty.png)

### Step 3: Configure LangSmith Alert with PagerDuty

<Info>
알림이 트리거된 후 한 시간 이내에 동일한 알림을 다시 받으려면 PagerDuty에서 알림으로 생성된 활성 incident를 해결해야 합니다.
</Info>

![PagerDuty Setup](/langsmith/images/pagerduty-setup.png)

1. LangSmith의 알림 설정에서 알림 섹션에서 **PagerDuty**를 선택합니다
2. 키 아이콘을 클릭하여 Integration Key를 Workspace secret으로 저장하거나 기존 Workspace secret을 선택합니다. 모범 사례로, Integration Key를 직접 추가하는 대신 Workspace Secret으로 저장하는 것을 권장합니다. 이렇게 하면 workspace의 여러 알림에서 동일한 키를 재사용할 수 있습니다.
3. 추가 알림 옵션을 구성합니다:
   * **Severity**: PagerDuty incident 우선순위에 매핑됩니다
4. **Send Test Alert**를 클릭하여 테스트 알림을 보냅니다
5. PagerDuty에서 incident가 트리거되고 관련 LangSmith 알림 정보가 포함되어 있는지 확인합니다

## Troubleshooting

PagerDuty에서 incident가 생성되지 않는 경우:

* LangSmith에 Integration Key가 올바르게 입력되었는지 확인합니다
* PagerDuty 서비스가 활성 상태이고 유지보수 모드가 아닌지 확인합니다
* PagerDuty 계정에 Events API v2가 활성화되어 있는지 확인합니다
* 알림 트리거가 PagerDuty에서 누락된 것으로 보이는 경우, 예상 트리거가 동일한 알림 규칙의 이전 트리거로부터 한 시간 이내에 발생했는지, 그리고 이전 알림으로 생성된 incident가 여전히 열려 있는지 확인합니다
* LangSmith 인스턴스가 방화벽 뒤에 있는 경우 네트워크 연결을 검토합니다
