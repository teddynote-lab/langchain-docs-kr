이 빠른 시작 가이드는 테스트 및 개발을 위해 LangGraph 애플리케이션을 로컬에서 설정하는 방법을 보여줍니다.

## 사전 요구 사항

시작하기 전에 [LangSmith](https://smith.langchain.com/settings)의 API key가 있는지 확인하세요 (무료 가입 가능).

## 1. LangGraph CLI 설치하기

<Tabs>
    <Tab title="Python server">
    ```shell
    # Python >= 3.11 is required.

    pip install -U "langgraph-cli[inmem]"
    ```
    </Tab>
    <Tab title="Node server">
    ```shell
    npx @langchain/langgraph-cli
    ```
    </Tab>
</Tabs>

## 2. LangGraph 앱 생성하기 🌱

[`new-langgraph-project-python` template](https://github.com/langchain-ai/new-langgraph-project) 또는 [`new-langgraph-project-js` template](https://github.com/langchain-ai/new-langgraphjs-project)에서 새 앱을 생성하세요. 이 template은 자신의 로직으로 확장할 수 있는 단일 노드 애플리케이션을 보여줍니다.

<Tabs>
    <Tab title="Python server">
    ```shell
    langgraph new path/to/your/app --template new-langgraph-project-python
    ```
    </Tab>
    <Tab title="Node server">
    ```shell
    langgraph new path/to/your/app --template new-langgraph-project-js
    ```
    </Tab>
</Tabs>

<Tip>
**추가 template**<br></br>
template을 지정하지 않고 [`langgraph new`](/langsmith/cli)를 사용하면 사용 가능한 template 목록에서 선택할 수 있는 대화형 메뉴가 표시됩니다.
</Tip>

## 3. 의존성 설치하기

새 LangGraph 앱의 루트에서 로컬 변경 사항이 서버에서 사용되도록 `edit` 모드로 의존성을 설치하세요:

<Tabs>
    <Tab title="Python server">
    ```shell
    cd path/to/your/app
    pip install -e .
    ```
    </Tab>
    <Tab title="Node server">
    ```shell
    cd path/to/your/app
    yarn install
    ```
    </Tab>
</Tabs>

## 4. `.env` 파일 생성하기

새 LangGraph 앱의 루트에서 [`.env.example`](/langsmith/application-structure#configuration-file)을 찾을 수 있습니다. 새 LangGraph 앱의 루트에 `.env` 파일을 생성하고 `.env.example` 파일의 내용을 복사하여 필요한 API key를 입력하세요:

```bash
LANGSMITH_API_KEY=lsv2...
```

## 5. LangGraph Server 실행하기 🚀

LangGraph API server를 로컬에서 시작하세요:

<Tabs>
    <Tab title="Python server">
    ```shell
    langgraph dev
    ```
    </Tab>
    <Tab title="Node server">
    ```shell
    npx @langchain/langgraph-cli dev
    ```
    </Tab>
</Tabs>

샘플 출력:

```
>    Ready!
>
>    - API: [http://localhost:2024](http://localhost:2024/)
>
>    - Docs: http://localhost:2024/docs
>
>    - Studio Web UI: https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024
```

[`langgraph dev`](/langsmith/cli) 명령은 인메모리 모드로 [LangGraph Server](/langsmith/langgraph-server)를 시작합니다. 이 모드는 개발 및 테스트 목적에 적합합니다.

<Tip>
프로덕션 사용을 위해서는 영구 스토리지 백엔드와 함께 LangGraph Server를 배포하세요. 자세한 내용은 LangSmith [호스팅 옵션](/langsmith/hosting)을 참조하세요.
</Tip>

## 6. API 테스트하기

<Tabs>
    <Tab title="Python SDK (async)">
    1. LangGraph Python SDK를 설치하세요:
      ```shell
      pip install langgraph-sdk
      ```
    2. assistant에게 메시지를 보내세요 (threadless run):
      ```python
      from langgraph_sdk import get_client
      import asyncio

      client = get_client(url="http://localhost:2024")

      async def main():
          async for chunk in client.runs.stream(
              None,  # Threadless run
              "agent", # Name of assistant. Defined in langgraph.json.
              input={
              "messages": [{
                  "role": "human",
                  "content": "What is LangGraph?",
                  }],
              },
          ):
              print(f"Receiving new event of type: {chunk.event}...")
              print(chunk.data)
              print("\n\n")

      asyncio.run(main())
      ```
    </Tab>
    <Tab title="Python SDK (sync)">
    1. LangGraph Python SDK를 설치하세요:
      ```shell
      pip install langgraph-sdk
      ```
    2. assistant에게 메시지를 보내세요 (threadless run):
      ```python
      from langgraph_sdk import get_sync_client

      client = get_sync_client(url="http://localhost:2024")

      for chunk in client.runs.stream(
          None,  # Threadless run
          "agent", # Name of assistant. Defined in langgraph.json.
          input={
              "messages": [{
                  "role": "human",
                  "content": "What is LangGraph?",
              }],
          },
          stream_mode="messages-tuple",
      ):
          print(f"Receiving new event of type: {chunk.event}...")
          print(chunk.data)
          print("\n\n")
      ```
    </Tab>
    <Tab title="Javascript SDK">
    1. LangGraph JS SDK를 설치하세요:
      ```shell
      npm install @langchain/langgraph-sdk
      ```
    2. assistant에게 메시지를 보내세요 (threadless run):
      ```js
      const { Client } = await import("@langchain/langgraph-sdk");

      // only set the apiUrl if you changed the default port when calling langgraph dev
      const client = new Client({ apiUrl: "http://localhost:2024"});

      const streamResponse = client.runs.stream(
          null, // Threadless run
          "agent", // Assistant ID
          {
              input: {
                  "messages": [
                      { "role": "user", "content": "What is LangGraph?"}
                  ]
              },
              streamMode: "messages-tuple",
          }
      );

      for await (const chunk of streamResponse) {
          console.log(`Receiving new event of type: ${chunk.event}...`);
          console.log(JSON.stringify(chunk.data));
          console.log("\n\n");
      }
      ```
    </Tab>
    <Tab title="Rest API">
    ```bash
    curl -s --request POST \
        --url "http://localhost:2024/runs/stream" \
        --header 'Content-Type: application/json' \
        --data "{
            \"assistant_id\": \"agent\",
            \"input\": {
                \"messages\": [
                    {
                        \"role\": \"human\",
                        \"content\": \"What is LangGraph?\"
                    }
                ]
            },
            \"stream_mode\": \"messages-tuple\"
        }"
    ```
    </Tab>
</Tabs>

## 다음 단계

이제 LangGraph 앱이 로컬에서 실행되고 있으므로 배포할 준비가 되었습니다:

**LangSmith의 호스팅 옵션을 선택하세요:**
- [**Cloud**](/langsmith/cloud): 가장 빠른 설정, 완전 관리형 (권장).
- [**Hybrid**](/langsmith/hybrid): <Tooltip tip="LangGraph Server와 에이전트가 실행되는 런타임 환경입니다.">Data plane</Tooltip>은 사용자의 클라우드에, <Tooltip tip="배포를 관리하기 위한 LangSmith UI 및 API입니다.">control plane</Tooltip>은 LangChain에서 관리.
- [**Self-hosted**](/langsmith/self-hosted): 사용자의 인프라에서 완전한 제어.

자세한 내용은 [호스팅 비교](/langsmith/hosting)를 참조하세요.

**그런 다음 앱을 배포하세요:**
- [Cloud 배포 빠른 시작](/langsmith/deployment-quickstart): 빠른 설정 가이드.
- [전체 Cloud 설정 가이드](/langsmith/deploy-to-cloud): 포괄적인 배포 문서.

**기능 살펴보기:**
- **[Studio](/langsmith/studio)**: Studio UI로 애플리케이션을 시각화하고, 상호작용하고, 디버그하세요. [Studio 빠른 시작](/langsmith/quick-start-studio)을 사용해보세요.
- **API References**: [LangGraph Server API](https://langchain-ai.github.io/langgraph/cloud/reference/api/api_ref/), [Python SDK](/langsmith/langgraph-python-sdk), [JS/TS SDK](/langsmith/langgraph-js-ts-sdk)
