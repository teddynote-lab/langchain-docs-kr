Agent Builder를 사용하면 자연어 아이디어를 프로덕션 에이전트로 전환할 수 있습니다. [deep-agents](https://github.com/langchain-ai/deepagents)를 기반으로 하며, <Tooltip tip="특정 순서로 작동하도록 설계된 미리 정해진 코드 경로입니다.">workflow 기반</Tooltip>이 아닙니다.

## Triggers

Trigger는 에이전트가 실행을 시작해야 하는 시점을 정의합니다. 에이전트를 외부 도구나 시간 기반 스케줄에 연결하여 메시지, 이메일 또는 반복 이벤트에 자동으로 응답하도록 할 수 있습니다.

다음 예시는 에이전트를 트리거하는 데 사용할 수 있는 일부 앱을 보여줍니다:

<CardGroup cols={3}>
  <Card title="Slack" icon="slack">
    특정 Slack 채널에서 메시지가 수신될 때 에이전트를 활성화합니다.
  </Card>
  <Card title="Gmail" icon="envelope">
    이메일이 수신될 때 에이전트를 트리거합니다.
  </Card>
  <Card title="Cron schedules" icon="clock">
    반복 작업을 위해 시간 기반 스케줄에 따라 에이전트를 실행합니다.
  </Card>
</CardGroup>

## Sub-agents

Agent Builder를 사용하면 메인 에이전트 내에서 sub-agent를 생성할 수 있습니다. Sub-agent는 더 큰 작업의 특정 부분을 처리하는 작고 특화된 에이전트입니다. 메인 에이전트와 조율하면서 자체 도구, 권한 또는 목표를 가지고 작동할 수 있습니다.

Sub-agent를 사용하면 작업을 집중적이고 재사용 가능한 구성 요소로 나누어 복잡한 시스템을 더 쉽게 구축할 수 있습니다. 이러한 모듈식 접근 방식은 에이전트를 체계적이고 확장 가능하며 유지 관리하기 쉽게 유지하는 데 도움이 됩니다.

다음은 프로젝트에서 sub-agent를 사용할 수 있는 몇 가지 방법입니다:

- 더 광범위한 workflow의 개별 부분 처리 (예: 데이터 검색, 요약 또는 포맷팅).
- 특화된 작업을 위해 다른 도구나 context window 사용.
- 독립적으로 실행되지만 결과를 메인 에이전트에 보고.

## Human in the loop

Human-in-the-loop 기능을 사용하면 에이전트 작업이 실행되기 전에 검토하고 승인할 수 있어 중요한 결정에 대한 제어권을 가질 수 있습니다.

### Interrupt 활성화

<Steps>
  <Step title="도구 선택">
    Agent Builder에서 에이전트를 구성할 때 사람의 감독을 추가하려는 도구를 선택합니다.
  </Step>
  <Step title="Interrupt 활성화">
    도구를 선택할 때 interrupt 옵션을 찾아 켭니다.
  </Step>
  <Step title="에이전트가 승인을 위해 일시 중지">
    에이전트는 해당 도구를 실행하기 전에 일시 중지하고 사람의 승인을 기다립니다.
  </Step>
</Steps>

### Interrupt 시 작업

에이전트가 interrupt 지점에 도달하면 다음 세 가지 작업 중 하나를 수행할 수 있습니다:

<CardGroup cols={3}>
  <Card title="Accept" icon="check">
    에이전트가 제안한 작업을 승인하고 계획대로 진행하도록 허용합니다.
  </Card>
  <Card title="Edit" icon="pen-to-square">
    에이전트가 계속 진행하기 전에 메시지나 매개변수를 수정합니다.
  </Card>
  <Card title="Send feedback" icon="comment">
    에이전트에게 피드백을 제공합니다.
  </Card>
</CardGroup>
