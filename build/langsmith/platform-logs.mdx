LangSmith에서 실행으로 생성된 trace를 볼 때, trace 뷰에서 직접 관련된 서버 로그에 액세스할 수 있습니다.

<Note>
trace에 대한 서버 로그 보기는 [Cloud SaaS](https://langchain-ai.github.io/langgra/langsmith/observability-concepts/deployment_options/#cloud-saas) 및 [완전 자체 호스팅](https://langchain-ai.github.io/langgra/langsmith/observability-concepts/deployment_options/#self-hosted-control-plane) 배포 옵션에서만 작동합니다.
</Note>

## trace 뷰에서 서버 로그 액세스하기

trace 뷰에서 **Run in Studio** 버튼 옆 오른쪽 상단의 **See Logs** 버튼을 사용하세요.

![](/langsmith/images/view-server-logs-button.png)

이 버튼을 클릭하면 LangSmith에서 관련 배포에 대한 서버 로그 뷰로 이동합니다.

## 서버 로그 뷰

서버 로그 뷰는 다음 두 가지 로그를 표시합니다:

- **LangGraph Server 자체의 운영 로그** - 내부 서버 작업, API 호출 및 시스템 이벤트
- **사용자 애플리케이션 로그** - 다음을 사용하여 그래프에 작성된 로그:
  - Python: `logging` 또는 `structlog` 라이브러리 사용
  - JavaScript: `@langchain/langgraph-sdk/logging`에서 재내보낸 Winston logger 사용:

```javascript
import { getLogger } from "@langchain/langgraph-sdk/logging";

const logger = getLogger();
logger.info("Your log message");
```

## trace ID로 로그 필터링하기

trace 뷰에서 이동하면 **Filters** 박스가 방금 본 trace의 Trace ID로 자동으로 미리 채워집니다.

이를 통해 특정 trace 실행과 관련된 로그만 빠르게 필터링하여 볼 수 있습니다.

![](/langsmith/images/lgp-server-logs-filters.png)
