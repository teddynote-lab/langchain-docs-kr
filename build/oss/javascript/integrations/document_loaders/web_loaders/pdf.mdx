이 노트북은 [WebPDFLoader](/oss/javascript/integrations/document_loaders/) 시작하기에 대한 간단한 개요를 제공합니다. 모든 WebPDFLoader 기능 및 구성에 대한 자세한 문서는 [API reference](https://api.js.langchain.com/classes/langchain_community_document_loaders_web_pdf.WebPDFLoader.html)를 참조하세요.

## 개요

### Integration 세부 정보

| Class | Package | Local | Serializable | PY support |
| :--- | :--- | :---: | :---: |  :---: |
| [WebPDFLoader](https://api.js.langchain.com/classes/langchain_community_document_loaders_web_pdf.WebPDFLoader.html) | [@langchain/community](https://api.js.langchain.com/modules/langchain_community_document_loaders_web_pdf.html) | ✅ | beta | ❌ |

### Loader 기능

| Source | Web Loader | Node Envs Only
| :---: | :---: | :---: |
| WebPDFLoader | ✅ | ❌ |

웹 환경에서 널리 사용되는 PDFLoader의 이 버전을 사용할 수 있습니다.
기본적으로 PDF 파일의 각 페이지마다 하나의 document가 생성되며, `splitPages` 옵션을 `false`로 설정하여 이 동작을 변경할 수 있습니다.

## 설정

`WebPDFLoader` document loader에 액세스하려면 `pdf-parse` 패키지와 함께 `@langchain/community` integration을 설치해야 합니다:

### Credentials

모델 호출에 대한 자동 추적을 원하시면 아래 주석을 해제하여 [LangSmith](https://docs.smith.langchain.com/) API key를 설정할 수 있습니다:

```bash
# export LANGSMITH_TRACING="true"
# export LANGSMITH_API_KEY="your-api-key"
```

### 설치

LangChain WebPDFLoader integration은 `@langchain/community` 패키지에 있습니다:

<CodeGroup>
```bash npm
npm install @langchain/community @langchain/core pdf-parse
```
```bash yarn
yarn add @langchain/community @langchain/core pdf-parse
```
```bash pnpm
pnpm add @langchain/community @langchain/core pdf-parse
```
</CodeGroup>

## 인스턴스화

이제 model 객체를 인스턴스화하고 document를 로드할 수 있습니다:

```typescript
import fs from "fs/promises";
import { WebPDFLoader } from "@langchain/community/document_loaders/web/pdf"

const nike10kPDFPath = "../../../../data/nke-10k-2023.pdf";

// Read the file as a buffer
const buffer = await fs.readFile(nike10kPDFPath);

// Create a Blob from the buffer
const nike10kPDFBlob = new Blob([buffer], { type: 'application/pdf' });

const loader = new WebPDFLoader(nike10kPDFBlob, {
  // required params = ...
  // optional params = ...
})
```

## 사용법, 커스텀 `pdfjs` 빌드

기본적으로 Node.js 및 최신 브라우저를 포함한 대부분의 환경과 호환되는 `pdf-parse`와 번들로 제공되는 `pdfjs` 빌드를 사용합니다. 더 최신 버전의 `pdfjs-dist`를 사용하거나 커스텀 빌드의 `pdfjs-dist`를 사용하려면, `PDFJS` 객체로 resolve되는 promise를 반환하는 커스텀 `pdfjs` 함수를 제공하여 사용할 수 있습니다.

다음 예제에서는 기본 빌드에 포함되지 않은 여러 polyfill을 포함하는 `pdfjs-dist`의 "legacy" 빌드([pdfjs docs](https://github.com/mozilla/pdf.js/wiki/Frequently-Asked-Questions#which-browsersenvironments-are-supported) 참조)를 사용합니다.

<CodeGroup>
```bash npm
npm install pdfjs-dist
```
```bash yarn
yarn add pdfjs-dist
```
```bash pnpm
pnpm add pdfjs-dist
```
</CodeGroup>


```typescript
import { WebPDFLoader } from "@langchain/community/document_loaders/web/pdf";

const blob = new Blob(); // e.g. from a file input

const customBuildLoader = new WebPDFLoader(blob, {
  // you may need to add `.then(m => m.default)` to the end of the import
  // @lc-ts-ignore
  pdfjs: () => import("pdfjs-dist/legacy/build/pdf.js"),
});
```

## 추가 공백 제거

PDF는 다양한 형태로 제공되므로 읽기가 어려울 수 있습니다. loader는 개별 텍스트 요소를 파싱하고 기본적으로 공백으로 결합하지만,
과도한 공백이 표시되는 경우 이것이 원하는 동작이 아닐 수 있습니다. 이 경우 다음과 같이 빈 문자열로 separator를 재정의할 수 있습니다:

```typescript
import { WebPDFLoader } from "@langchain/community/document_loaders/web/pdf";

// new Blob(); e.g. from a file input
const eliminatingExtraSpacesLoader = new WebPDFLoader(new Blob(), {
  parsedItemSeparator: "",
});
```
