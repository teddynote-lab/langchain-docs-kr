이 가이드는 LangChain을 사용하여 ByteDanceDoubao [embedding models](/oss/javascript/integrations/text_embedding)를 시작하는 데 도움을 드립니다. `ByteDanceDoubaoEmbeddings`의 기능 및 구성 옵션에 대한 자세한 문서는 [API reference](https://api.js.langchain.com/classes/_langchain_community.embeddings_bytedance_doubao.ByteDanceDoubaoEmbeddings.html)를 참조하세요.

## Setup

[ARK API key](https://console.volcengine.com/ark/region:ark+cn-beijing/apiKey)에 가입하고 `ARK_API_KEY`라는 환경 변수로 설정해야 합니다. 그런 다음 embedding models를 위한 [entrypoint를 생성](https://console.volcengine.com/ark/region:ark+cn-beijing/endpoint)하고, entrypoint의 이름을 `model`로 사용해야 합니다.

그 다음, [`@langchain/community`](https://www.npmjs.com/package/@langchain/community) package를 설치해야 합니다.

### Credentials

모델 호출에 대한 자동 추적을 원하시면 아래 주석을 해제하여 [LangSmith](https://docs.smith.langchain.com/) API key를 설정할 수 있습니다:

```bash
# export LANGCHAIN_TRACING_V2="true"
# export LANGCHAIN_API_KEY="your-api-key"
```

### Installation

LangChain ByteDanceDoubaoEmbeddings integration은 `@langchain/community` package에 있습니다:

<CodeGroup>
```bash npm
npm install @langchain/community
```
```bash yarn
yarn add @langchain/community
```
```bash pnpm
pnpm add @langchain/community
```
</CodeGroup>

## Instantiation

이제 model object를 인스턴스화하고 텍스트를 embed할 수 있습니다:

```typescript
import { ByteDanceDoubaoEmbeddings } from "@langchain/community/embeddings/bytedance_doubao";

const embeddings = new ByteDanceDoubaoEmbeddings({
  model: 'ep-xxx-xxx' // your entrypoint's name
});
```

## Indexing and Retrieval

Embedding models는 데이터 인덱싱과 나중에 검색하는 과정 모두에서 retrieval-augmented generation (RAG) 플로우에 자주 사용됩니다. 더 자세한 지침은 [**Learn** 탭](/oss/javascript/learn/)의 RAG 튜토리얼을 참조하세요.

아래에서는 위에서 초기화한 `embeddings` object를 사용하여 데이터를 인덱싱하고 검색하는 방법을 확인할 수 있습니다. 이 예제에서는 데모 [`MemoryVectorStore`](/oss/javascript/integrations/vectorstores/memory)를 사용하여 샘플 문서를 인덱싱하고 검색합니다.

```typescript
// Create a vector store with a sample text
import { MemoryVectorStore } from "@langchain/classic/vectorstores/memory";

const text = "LangChain is the framework for building context-aware reasoning applications";

const vectorstore = await MemoryVectorStore.fromDocuments(
  [{ pageContent: text, metadata: {} }],
  embeddings,
);

// Use the vector store as a retriever that returns a single document
const retriever = vectorstore.asRetriever(1);

// Retrieve the most similar text
const retrievedDocuments = await retriever.invoke("What is LangChain?");

retrievedDocuments[0].pageContent;
```

```output
LangChain is the framework for building context-aware reasoning applications
```

## Direct Usage

내부적으로 vectorstore와 retriever 구현은 `embeddings.embedDocument(...)`와 `embeddings.embedQuery(...)`를 호출하여 `fromDocuments`와 retriever의 `invoke` 작업에 사용되는 텍스트에 대한 embeddings를 생성합니다.

이러한 메서드를 직접 호출하여 자신의 사용 사례에 맞는 embeddings를 얻을 수 있습니다.

### Embed single texts

검색을 위해 `embedQuery`로 쿼리를 embed할 수 있습니다. 이는 쿼리에 특화된 vector 표현을 생성합니다:

```typescript
const singleVector = await embeddings.embedQuery(text);

console.log(singleVector.slice(0, 100));
```

### Embed multiple texts

인덱싱을 위해 `embedDocuments`로 여러 텍스트를 embed할 수 있습니다. 이 메서드에 사용되는 내부 구현은 쿼리 embedding과 다를 수 있습니다(반드시 그런 것은 아닙니다):

```typescript
const text2 = "LangGraph is a library for building stateful, multi-actor applications with LLMs";

const vectors = await embeddings.embedDocuments([text, text2]);

console.log(vectors[0].slice(0, 100));
console.log(vectors[1].slice(0, 100));
```
