## Setup

Cohere embedding model에 액세스하려면 Cohere 계정을 생성하고 API key를 받은 다음 `@langchain/cohere` integration package를 설치해야 합니다.

### Credentials

[cohere.com](https://cohere.com)으로 이동하여 `Cohere`에 가입하고 API key를 생성하세요. 완료한 후 `COHERE_API_KEY` environment variable을 설정하세요:

```bash
export COHERE_API_KEY="your-api-key"
```

model 호출에 대한 자동 추적을 원하시면 아래 주석을 해제하여 [LangSmith](https://docs.smith.langchain.com/) API key를 설정할 수도 있습니다:

```bash
# export LANGSMITH_TRACING="true"
# export LANGSMITH_API_KEY="your-api-key"
```

### Installation

LangChain CohereEmbeddings integration은 `@langchain/cohere` package에 있습니다:

<CodeGroup>
```bash npm
npm install @langchain/cohere @langchain/core
```
```bash yarn
yarn add @langchain/cohere @langchain/core
```
```bash pnpm
pnpm add @langchain/cohere @langchain/core
```
</CodeGroup>

## Instantiation

이제 model object를 인스턴스화하고 chat completion을 생성할 수 있습니다:

```typescript
import { CohereEmbeddings } from "@langchain/cohere";

const embeddings = new CohereEmbeddings({
  apiKey: "YOUR-API-KEY", // In Node.js defaults to process.env.COHERE_API_KEY
  batchSize: 48, // Default value if omitted is 48. Max value is 96
  model: "embed-english-v3.0",
});
```

### Custom client for Cohere on Azure, Cohere on AWS Bedrock, and Standalone Cohere Instance

커스텀 `CohereClient`를 인스턴스화하여 ChatCohere constructor에 전달할 수 있습니다.

**참고:** 커스텀 client가 제공되면 `COHERE_API_KEY` environment variable과 constructor의 apiKey parameter가 모두 무시됩니다

```typescript
import { CohereEmbeddings } from "@langchain/cohere";
import { CohereClient } from "cohere-ai";

const client = new CohereClient({
  token: "<your-api-key>",
  environment: "<your-cohere-deployment-url>", //optional
  // other params
});

const embeddingsWithCustomClient = new CohereEmbeddings({
  client,
  // other params...
});
```

## Indexing and Retrieval

Embedding model은 데이터 인덱싱과 나중에 검색하는 과정 모두에서 retrieval-augmented generation (RAG) flow에 자주 사용됩니다. 더 자세한 지침은 [**Learn** 탭](/oss/javascript/learn/)의 RAG 튜토리얼을 참조하세요.

아래에서는 위에서 초기화한 `embeddings` object를 사용하여 데이터를 인덱싱하고 검색하는 방법을 확인할 수 있습니다. 이 예제에서는 데모 [`MemoryVectorStore`](/oss/javascript/integrations/vectorstores/memory)를 사용하여 샘플 문서를 인덱싱하고 검색합니다.

```typescript
// Create a vector store with a sample text
import { MemoryVectorStore } from "@langchain/classic/vectorstores/memory";

const text = "LangChain is the framework for building context-aware reasoning applications";

const vectorstore = await MemoryVectorStore.fromDocuments(
  [{ pageContent: text, metadata: {} }],
  embeddings,
);

// Use the vector store as a retriever that returns a single document
const retriever = vectorstore.asRetriever(1);

// Retrieve the most similar text
const retrievedDocuments = await retriever.invoke("What is LangChain?");

retrievedDocuments[0].pageContent;
```

```output
LangChain is the framework for building context-aware reasoning applications
```

## Direct Usage

내부적으로 vectorstore와 retriever 구현은 `embeddings.embedDocument(...)`와 `embeddings.embedQuery(...)`를 호출하여 `fromDocuments`와 retriever의 `invoke` 작업에 각각 사용되는 텍스트에 대한 embedding을 생성합니다.

이러한 메서드를 직접 호출하여 자신의 사용 사례에 맞는 embedding을 얻을 수 있습니다.

const singleVector = await embeddings.embedQuery(text);

console.log(singleVector.slice(0, 100));

### Embed multiple texts

인덱싱을 위해 `embedDocuments`로 여러 텍스트를 embedding할 수 있습니다. 이 메서드에 사용되는 내부 구현은 query embedding과 다를 수 있습니다(반드시 다를 필요는 없습니다):

```typescript
const text2 = "LangGraph is a library for building stateful, multi-actor applications with LLMs";

const vectors = await embeddings.embedDocuments([text, text2]);

console.log(vectors[0].slice(0, 100));
console.log(vectors[1].slice(0, 100));
```
