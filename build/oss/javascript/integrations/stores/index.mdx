## 개요

LangChain은 키로 데이터를 저장하고 검색하기 위한 key-value store interface를 제공합니다. LangChain의 key-value store interface는 주로 [embeddings](/oss/javascript/integrations/text_embedding)를 캐싱하는 데 사용됩니다.

## Interface

모든 [`BaseStores`](https://api.js.langchain.com/classes/langchain_core.stores.BaseStore.html)는 **generic**이며 다음 interface를 지원합니다. 여기서 `K`는 key type을, `V`는 value type을 나타냅니다:

- `mget(keys: K[]): Promise<(V | undefined)[]>`: 여러 key에 대한 value를 가져오며, key가 존재하지 않으면 `undefined`를 반환합니다
- `mset(keyValuePairs: [K, V][]): Promise<void>`: 여러 key에 대한 value를 설정합니다
- `mdelete(keys: K[]): Promise<void>`: 여러 key를 삭제합니다
- `yieldKeys(prefix?: string): AsyncGenerator<K | string>`: store의 모든 key를 비동기적으로 생성하며, 선택적으로 prefix로 필터링할 수 있습니다

interface의 generic 특성을 통해 key와 value에 다양한 type을 사용할 수 있습니다. 예를 들어, `BaseStore<string, BaseMessage>`는 string key로 message를 저장하고, `BaseStore<string, number[]>`는 숫자 배열을 저장합니다.

<Note>
Base store는 효율성을 위해 **여러** key-value 쌍을 한 번에 처리하도록 설계되었습니다. 이를 통해 네트워크 왕복 횟수를 줄이고 기본 store에서 더 효율적인 batch 작업을 수행할 수 있습니다.
</Note>
