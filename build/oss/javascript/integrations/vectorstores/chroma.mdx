[Chroma](https://docs.trychroma.com/getting-started)는 개발자 생산성과 만족도에 중점을 둔 AI 네이티브 오픈소스 vector database입니다. Chroma는 Apache 2.0 라이선스로 제공됩니다.

이 가이드는 Chroma [`vector stores`](/oss/javascript/integrations/vectorstores) 시작하기에 대한 간단한 개요를 제공합니다. 모든 `Chroma` 기능 및 구성에 대한 자세한 문서는 [API reference](https://api.js.langchain.com/classes/langchain_community_vectorstores_chroma.Chroma.html)를 참조하세요.

<Info>
**Chroma Cloud**

Chroma Cloud는 서버리스 vector 및 full-text search를 지원합니다. 매우 빠르고, 비용 효율적이며, 확장 가능하고 간편합니다. 30초 이내에 DB를 생성하고 $5의 무료 크레딧으로 사용해보세요.

[Chroma Cloud 시작하기](https://trychroma.com/signup)

</Info>

## Setup

Chroma vector store를 사용하려면 `@langchain/community` integration package와 peer dependency로 [Chroma JS SDK](https://www.npmjs.com/package/chromadb)를 설치해야 합니다.

이 가이드는 [OpenAI embeddings](/oss/javascript/integrations/text_embedding/openai)도 사용하므로 `@langchain/openai` integration package를 설치해야 합니다. 원하는 경우 [다른 지원되는 embeddings models](/oss/javascript/integrations/text_embedding)을 사용할 수도 있습니다.

<CodeGroup>
```bash npm
npm install @langchain/community @langchain/openai @langchain/core chromadb
```
```bash yarn
yarn add @langchain/community @langchain/openai @langchain/core chromadb
```
```bash pnpm
pnpm add @langchain/community @langchain/openai @langchain/core chromadb
```
</CodeGroup>

Chroma를 로컬에서 실행하려면 `chromadb` package와 함께 제공되는 Chroma CLI를 사용하여 [로컬 Chroma server를 실행](https://docs.trychroma.com/docs/cli/run)할 수 있습니다:

```
chroma run
```

공식 Chroma image를 사용하여 Docker에서 server를 실행할 수도 있습니다:

```
docker pull chromadb/chroma
docker run -p 8000:8000 chromadb/chroma
```

### Credentials

Chroma를 로컬에서 실행하는 경우 자격 증명을 제공할 필요가 없습니다.

[Chroma Cloud](https://trychroma.com/signup) 사용자인 경우 `CHROMA_TENANT`, `CHROMA_DATABASE`, `CHROMA_API_KEY` environment variables를 설정하세요.

Chroma CLI가 이를 설정할 수 있습니다. 먼저 CLI를 통해 [login](https://docs.trychroma.com/docs/cli/login)한 다음 [`connect` command](https://docs.trychroma.com/docs/cli/db)를 사용하세요:

```
chroma db connect [db_name] --env-file
```

이 가이드에서 OpenAI embeddings를 사용하는 경우 OpenAI key도 설정해야 합니다:

```typescript
process.env.OPENAI_API_KEY = "YOUR_API_KEY";
```

model 호출에 대한 자동 추적을 원하는 경우 아래 주석을 해제하여 [LangSmith](https://docs.smith.langchain.com/) API key를 설정할 수도 있습니다:

```typescript
// process.env.LANGSMITH_TRACING="true"
// process.env.LANGSMITH_API_KEY="your-api-key"
```

## Instantiation

### Setup your embedding function

먼저 embedding function을 선택하세요. 여기서는 `OpenAIEmbeddings`를 사용합니다:

```python
import { OpenAIEmbeddings } from "@langchain/openai";

const embeddings = new OpenAIEmbeddings({
  model: "text-embedding-3-small",
});
```

### Running Locally

간단한 `Chroma` instantiation은 `http://localhost:8000`에서 로컬로 실행되는 Chroma server에 연결됩니다:

```python
import { Chroma } from "@langchain/community/vectorstores/chroma";

const vectorStore = new Chroma(embeddings, {
  collectionName: "a-test-collection"
});
```

다른 구성으로 Chroma server를 실행하는 경우 `host`, `port` 및 `ssl`을 사용한 연결 여부를 지정할 수 있습니다:

```python
import { Chroma } from "@langchain/community/vectorstores/chroma";

const vectorStore = new Chroma(embeddings, {
  collectionName: "a-test-collection",
  host: "your-host-address",
  port: 8080
});
```

### Chroma Cloud

Chroma Cloud에 연결하려면 `tenant`, `database`, `chromaCloudAPIKey`를 제공하세요:

```python
import { Chroma } from "@langchain/community/vectorstores/chroma";

const vectorStore = new Chroma(embeddings, {
  collectionName: "a-test-collection",
  chromaCloudAPIKey: process.env.CHROMA_API_KEY,
  clientParams: {
    host: "api.trychroma.com",
    port: 8000,
    ssl: true,
    tenant: process.env.CHROMA_TENANT,
    database: process.env.CHROMA_DATABASE,
  },
});
```

## Manage vector store

### Add items to vector store

```python
import type { Document } from "@langchain/core/documents";

const document1: Document = {
  pageContent: "The powerhouse of the cell is the mitochondria",
  metadata: { source: "https://example.com" }
};

const document2: Document = {
  pageContent: "Buildings are made out of brick",
  metadata: { source: "https://example.com" }
};

const document3: Document = {
  pageContent: "Mitochondria are made out of lipids",
  metadata: { source: "https://example.com" }
};

const document4: Document = {
  pageContent: "The 2024 Olympics are in Paris",
  metadata: { source: "https://example.com" }
}

const documents = [document1, document2, document3, document4];

await vectorStore.addDocuments(documents, { ids: ["1", "2", "3", "4"] });
```

### Delete items from vector store

다음과 같이 id로 Chroma에서 documents를 삭제할 수 있습니다:

```python
await vectorStore.delete({ ids: ["4"] });
```

## Query vector store

vector store가 생성되고 관련 documents가 추가되면 chain 또는 agent를 실행하는 동안 쿼리하고 싶을 것입니다.

### Query directly

간단한 similarity search는 다음과 같이 수행할 수 있습니다:

```python
const filter = { source: "https://example.com" };

const similaritySearchResults = await vectorStore.similaritySearch("biology", 2, filter);

for (const doc of similaritySearchResults) {
  console.log(`* ${doc.pageContent} [${JSON.stringify(doc.metadata, null)}]`);
}
```

Chroma filter syntax에 대한 자세한 내용은 [이 페이지](https://docs.trychroma.com/guides#filtering-by-metadata)를 참조하세요.

similarity search를 실행하고 해당 scores를 받으려면 다음을 실행할 수 있습니다:

```python
const similaritySearchWithScoreResults = await vectorStore.similaritySearchWithScore("biology", 2, filter)

for (const [doc, score] of similaritySearchWithScoreResults) {
  console.log(`* [SIM=${score.toFixed(3)}] ${doc.pageContent} [${JSON.stringify(doc.metadata)}]`);
}
```

### Query by turning into retriever

vector store를 [retriever](/oss/javascript/langchain/retrieval)로 변환하여 chains에서 더 쉽게 사용할 수도 있습니다.

```python
const retriever = vectorStore.asRetriever({
  // Optional filter
  filter: filter,
  k: 2,
});
await retriever.invoke("biology");
```
