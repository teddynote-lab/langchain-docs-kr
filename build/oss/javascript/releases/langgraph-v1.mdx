**LangGraph v1은 에이전트 런타임의 안정성에 중점을 둔 릴리스입니다.** 핵심 graph API와 실행 모델은 변경되지 않은 채로 유지하면서, 타입 안전성, 문서, 개발자 경험을 개선했습니다.

[LangChain v1](/oss/javascript/releases/langchain-v1)과 함께 작동하도록 설계되었으며 (LangChain의 `createAgent`는 LangGraph 위에 구축됨), 높은 수준에서 시작하여 필요할 때 세밀한 제어로 전환할 수 있습니다.

    </Card>
    <Card title="기본적으로 제공되는 신뢰성" icon="database">

    <Card title="LangChain v1과 원활한 통합" icon="link">

    </Card>

## `createReactAgent`의 지원 중단

LangGraph의 `createReactAgent` 사전 빌드는 LangChain의 `createAgent`를 위해 지원 중단되었습니다. 더 간단한 인터페이스를 제공하며, 미들웨어 도입을 통해 더 큰 커스터마이징 가능성을 제공합니다.

* 새로운 `createAgent` API에 대한 정보는 [LangChain v1 릴리스 노트](/oss/javascript/releases/langchain-v1#createagent)를 참조하세요.
* `createReactAgent`에서 `createAgent`로 마이그레이션하는 방법은 [LangChain v1 마이그레이션 가이드](/oss/javascript/migrate/langchain-v1#createagent)를 참조하세요.

## 타입이 지정된 interrupt

[`StateGraph`](https://langchain-ai.github.io/langgraphjs/reference/classes/langgraph.StateGraph.html)는 이제 생성자에서 interrupt 타입의 맵을 받아 graph 내에서 사용할 수 있는 interrupt 타입을 더 엄격하게 제한합니다.

```typescript expandable
import { StateGraph, MemorySaver, interrupt } from "@langchain/langgraph";
import * as z from "zod";

const stateSchema = z.object({
  foo: z.string(),
})

const graphConfig = {
  interrupts: {
    // Define a simple interrupt that accepts a reason and returns messages
    simple: interrupt<{ reason: string }, { messages: string[] }>, // [!code highlight]
    // Define a complex interrupt with the same signature
    complex: interrupt<{ reason: string }, { messages: string[] }>, // [!code highlight]
  }
}

const checkpointer = new MemorySaver();

const graph = new StateGraph(stateSchema, graphConfig)
  .addNode("node", async (state, runtime) => {
    // Trigger the simple interrupt with a reason
    const response = runtime.interrupt.simple({ reason: "test" });
    // Return the interrupt response as the new state
    return { foo: response };
  })
  // Compile the graph with the checkpointer
  .compile({ checkpointer });

// Invoke the graph with initial state
const result = await graph.invoke({ foo: "test" });

// Access the interrupt data
if (graph.isInterrupted(result)) {
  console.log(result.__interrupt__.messages);
}
```

interrupt에 대한 자세한 내용은 [Interrupts](/oss/javascript/langgraph/interrupts) 문서를 참조하세요.

## Frontend SDK 개선 사항

LangGraph v1은 프론트엔드에서 LangGraph 애플리케이션과 상호작용할 때 몇 가지 개선 사항을 제공합니다.

### Event stream encoding

저수준 `toLangGraphEventStream` 헬퍼가 제거되었습니다. 스트리밍 응답은 이제 SDK에서 기본적으로 처리되며, `graph.stream`에 `encoding` 형식을 전달하여 wire 형식을 선택할 수 있습니다. 이를 통해 UI 로직을 변경하지 않고도 SSE와 일반 JSON 응답 간에 쉽게 전환할 수 있습니다.

자세한 내용은 [마이그레이션 가이드](/oss/javascript/migrate/langgraph-v1#stream-encoding)를 참조하세요.

### `useStream`의 커스텀 transport

React `useStream` hook은 이제 플러그 가능한 transport를 지원하므로 UI 코드를 변경하지 않고도 네트워크 레이어를 더 세밀하게 제어할 수 있습니다.

```typescript
const stream = useStream({
  transport: new FetchStreamTransport({
    apiUrl: "http://localhost:2024",
  }),
});
```

hook을 통합하고 커스터마이징하는 방법을 알아보세요: [LangGraph를 React 애플리케이션에 통합하기](/oss/javascript/langgraph/ui).
