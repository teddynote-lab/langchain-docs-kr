[Google Cloud](https://cloud.google.com/), [Google Gemini](https://ai.google.dev/gemini-api/docs) 및 기타 Google 제품과의 모든 LangChain 통합입니다.

1. **Google Generative AI (Gemini API & AI Studio)**: Gemini API를 통해 Google Gemini 모델에 직접 액세스합니다. 빠른 프로토타이핑을 위해 [Google AI Studio](https://aistudio.google.com/)를 사용하고 `langchain-google-genai` 패키지로 빠르게 시작하세요. 개인 개발자에게는 이것이 가장 좋은 시작점입니다.
2. **Google Cloud (Vertex AI & 기타 서비스)**: [Google Cloud Platform](https://cloud.google.com/)을 통해 Gemini 모델, Vertex AI Model Garden 및 다양한 클라우드 서비스(데이터베이스, 스토리지, 문서 AI 등)에 액세스합니다. Vertex AI 모델에는 `langchain-google-vertexai` 패키지를, 기타 클라우드 서비스에는 특정 패키지(예: `langchain-google-cloud-sql-pg`, `langchain-google-community`)를 사용하세요. 이미 Google Cloud를 사용하고 있거나 MLOps, 특정 모델 튜닝 또는 엔터프라이즈 지원과 같은 엔터프라이즈 기능이 필요한 개발자에게 이상적입니다.

차이점에 대한 자세한 내용은 [Gemini API에서 Vertex AI로 마이그레이션](https://ai.google.dev/gemini-api/docs/migrate-to-cloud)에 대한 Google 가이드를 참조하세요.

Gemini 모델 및 Vertex AI 플랫폼용 통합 패키지는 [`langchain-google`](https://github.com/langchain-ai/langchain-google) 저장소에서 유지 관리됩니다. 다른 Google API 및 서비스와의 다양한 LangChain 통합은 [googleapis](https://github.com/googleapis?q=langchain-&type=all&language=&sort=) Github 조직 및 `langchain-google-community` 패키지에서 찾을 수 있습니다.

## Google Generative AI (Gemini API & AI Studio)

빠른 개발 및 실험에 가장 적합한 Gemini API를 사용하여 Google Gemini 모델에 직접 액세스합니다. Gemini 모델은 [Google AI Studio](https://aistudio.google.com/)에서 사용할 수 있습니다.

<CodeGroup>
    ```bash pip
    pip install -U langchain-google-genai
    ```

    ```bash uv
    uv add langchain-google-genai
    ```
</CodeGroup>

무료로 시작하고 [Google AI Studio](https://aistudio.google.com/app/apikey)에서 API 키를 받으세요.

```bash
export GOOGLE_API_KEY="YOUR_API_KEY"
```

### Chat models

`ChatGoogleGenerativeAI` 클래스를 사용하여 Gemini 모델과 상호 작용합니다. 자세한 내용은 [이 가이드](/oss/python/integrations/chat/google_generative_ai)를 참조하세요.

```python
from langchain_google_genai import ChatGoogleGenerativeAI
from langchain.messages import HumanMessage

llm = ChatGoogleGenerativeAI(model="gemini-2.5-flash")

# Simple text invocation
result = llm.invoke("Sing a ballad of LangChain.")
print(result.content)

# Multimodal invocation with gemini-pro-vision
message = HumanMessage(
    content=[
        {
            "type": "text",
            "text": "What's in this image?",
        },
        {"type": "image_url", "image_url": "https://picsum.photos/seed/picsum/200/300"},
    ]
)
result = llm.invoke([message])
print(result.content)
```

`image_url`은 공개 URL, GCS URI(`gs://...`), 로컬 파일 경로, base64로 인코딩된 이미지 문자열(`data:image/png;base64,...`) 또는 PIL Image 객체일 수 있습니다.


### Embedding Models

`GoogleGenerativeAIEmbeddings` 클래스를 사용하여 `gemini-embedding-001`과 같은 모델로 텍스트 임베딩을 생성합니다.

[사용 예제](/oss/python/integrations/text_embedding/google_generative_ai)를 참조하세요.

```python
from langchain_google_genai import GoogleGenerativeAIEmbeddings

embeddings = GoogleGenerativeAIEmbeddings(model="models/gemini-embedding-001")
vector = embeddings.embed_query("What are embeddings?")
print(vector[:5])
```

## Google Cloud

Vertex AI 및 특정 클라우드 통합을 통해 Gemini 모델, Vertex AI Model Garden 및 기타 Google Cloud 서비스에 액세스합니다.

Vertex AI 모델에는 `langchain-google-vertexai` 패키지가 필요합니다. 다른 서비스에는 `langchain-google-community`, `langchain-google-cloud-sql-pg` 등과 같은 추가 패키지가 필요할 수 있습니다.

<CodeGroup>
    ```bash pip
    pip install langchain-google-vertexai
    # pip install langchain-google-community[...] # For other services
    ```

    ```bash uv
    uv add langchain-google-vertexai
    # uv add langchain-google-community[...] # For other services
    ```
</CodeGroup>

Google Cloud 통합은 일반적으로 Application Default Credentials (ADC)를 사용합니다. 설정 지침은 [Google Cloud 인증 문서](https://cloud.google.com/docs/authentication)를 참조하세요(예: `gcloud auth application-default login` 사용).

#### Document AI

>[Google Cloud Document AI](https://cloud.google.com/document-ai/docs/overview)는 문서의 비구조화 데이터를 구조화된 데이터로 변환하여 이해, 분석 및 소비를 더 쉽게 만드는 Google Cloud 서비스입니다.

[`GCS` 버킷을 설정하고 자체 OCR 프로세서를 생성](https://cloud.google.com/document-ai/docs/create-processor)해야 합니다.
`GCS_OUTPUT_PATH`는 GCS의 폴더 경로(`gs://`로 시작)여야 하며
프로세서 이름은 `projects/PROJECT_NUMBER/locations/LOCATION/processors/PROCESSOR_ID`와 같아야 합니다.
프로그래밍 방식으로 가져오거나 Google Cloud Console의 `Processor details` 탭의 `Prediction endpoint` 섹션에서 복사할 수 있습니다.

### MCP Toolbox

[MCP Toolbox](https://github.com/googleapis/genai-toolbox)는 [Cloud SQL](https://cloud.google.com/sql/docs) 및 [AlloyDB](https://cloud.google.com/alloydb/docs/overview)와 같은 Google Cloud의 데이터베이스를 포함하여 데이터베이스에 연결하는 간단하고 효율적인 방법을 제공합니다. MCP Toolbox를 사용하면 데이터베이스를 LangChain과 원활하게 통합하여 강력한 데이터 기반 애플리케이션을 구축할 수 있습니다.

#### Installation

시작하려면 [Toolbox 서버 및 클라이언트를 설치](https://github.com/googleapis/genai-toolbox/releases/)하세요.

도구를 정의하기 위해 `tools.yaml`을 [구성](https://googleapis.github.io/genai-toolbox/getting-started/configure/)한 다음 toolbox를 실행하여 서버를 시작합니다:

```bash
toolbox --tools-file "tools.yaml"
```

그런 다음 Toolbox 클라이언트를 설치합니다:

<CodeGroup>
    ```bash pip
    pip install toolbox-langchain
    ```

    ```bash uv
    uv add toolbox-langchain
    ```
</CodeGroup>

#### Getting Started

다음은 MCP Toolbox를 사용하여 데이터베이스에 연결하는 방법에 대한 간단한 예제입니다:

```python
from toolbox_langchain import ToolboxClient

async with ToolboxClient("http://127.0.0.1:5000") as client:

    tools = client.load_toolset()
```
