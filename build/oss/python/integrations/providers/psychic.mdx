>[Psychic](https://www.psychic.dev/)은 `Notion`, `Zendesk`,
> `Confluence`, `Google Drive`와 같은 SaaS 도구와 OAuth를 통해 통합하고
> 이러한 애플리케이션의 문서를 SQL 또는 vector database로 동기화하는 플랫폼입니다. 비구조화된 데이터를 위한 Plaid와 같다고 생각하시면 됩니다.

## Installation and Setup

<CodeGroup>
```bash pip
pip install psychicapi
```

```bash uv
uv add psychicapi
```
</CodeGroup>

Psychic은 설정이 간단합니다 - `react` 라이브러리를 import하고 [Psychic dashboard](https://dashboard.psychic.dev/)에서 받은 `Sidekick API` key로 구성하면 됩니다. 애플리케이션을 연결하면
dashboard에서 이러한 연결을 확인하고 서버 측 라이브러리를 사용하여 데이터를 검색할 수 있습니다.

1. [dashboard](https://dashboard.psychic.dev/)에서 계정을 생성합니다.
2. [react library](https://docs.psychic.dev/sidekick-link)를 사용하여 프론트엔드 react 앱에 Psychic link modal을 추가합니다. 이를 사용하여 SaaS 앱을 연결합니다.
3. 연결을 생성한 후에는 [example notebook](/oss/python/integrations/document_loaders/psychic)을 참고하여 `PsychicLoader`를 사용할 수 있습니다.


## 다른 Document Loader 대비 장점

1.	**Universal API:** 모든 SaaS 앱에 대해 OAuth flow를 구축하고 API를 학습하는 대신, Psychic을 한 번 통합하고 universal API를 활용하여 데이터를 검색할 수 있습니다.
2.	**Data Syncs:** 고객의 SaaS 앱에 있는 데이터는 빠르게 오래될 수 있습니다. Psychic을 사용하면 webhook을 구성하여 문서를 일일 또는 실시간으로 최신 상태로 유지할 수 있습니다.
3.	**Simplified OAuth:** Psychic은 OAuth를 end-to-end로 처리하므로 각 통합에 대한 OAuth client를 생성하고, access token을 최신 상태로 유지하며, OAuth redirect 로직을 처리하는 데 시간을 소비할 필요가 없습니다.
