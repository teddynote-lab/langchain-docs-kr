## 설정

IBM WatsonxAI 모델에 액세스하려면 IBM watsonx.ai 계정을 생성하고, API 키 또는 기타 유형의 자격 증명을 받고, `@langchain/community` integration 패키지를 설치해야 합니다.

### 자격 증명

[IBM Cloud](https://cloud.ibm.com/login)로 이동하여 IBM watsonx.ai에 가입하고 API 키를 생성하거나 아래와 같이 다른 인증 형식을 제공하세요.

#### IAM 인증

```bash
export WATSONX_AI_AUTH_TYPE=iam
export WATSONX_AI_APIKEY=<YOUR-APIKEY>
```

#### Bearer token 인증

```bash
export WATSONX_AI_AUTH_TYPE=bearertoken
export WATSONX_AI_BEARER_TOKEN=<YOUR-BEARER-TOKEN>
```

#### IBM watsonx.ai software 인증

```bash
export WATSONX_AI_AUTH_TYPE=cp4d
export WATSONX_AI_USERNAME=<YOUR_USERNAME>
export WATSONX_AI_PASSWORD=<YOUR_PASSWORD>
export WATSONX_AI_URL=<URL>
```

이러한 값들이 환경 변수에 설정되고 객체가 초기화되면 인증이 자동으로 진행됩니다.

인증은 새 인스턴스에 이러한 값을 매개변수로 전달하여 수행할 수도 있습니다.

## IAM 인증

```typescript
import { WatsonxLLM } from "@langchain/community/llms/ibm";

const props = {
  version: "YYYY-MM-DD",
  serviceUrl: "<SERVICE_URL>",
  projectId: "<PROJECT_ID>",
  watsonxAIAuthType: "iam",
  watsonxAIApikey: "<YOUR-APIKEY>",
};
const instance = new WatsonxLLM(props);
```

모델 호출에 대한 자동 추적을 원하시면 아래 주석을 해제하여 [LangSmith](https://docs.smith.langchain.com/) API 키를 설정할 수도 있습니다:

```bash

# export LANGSMITH_API_KEY="your-api-key"
```

## 인스턴스화

이제 모델 객체를 인스턴스화하고 채팅 완성을 생성할 수 있습니다:

```javascript
import { WatsonxLLM } from "@langchain/community/llms/ibm";

const props = {
  decoding_method: "sample",
  maxNewTokens: 100,
  minNewTokens: 1,
  temperature: 0.5,
  topK: 50,
  topP: 1,
};
const instance = new WatsonxLLM({
  version: "YYYY-MM-DD",
  serviceUrl: process.env.API_URL,
  projectId: "<PROJECT_ID>",
  // spaceId: "<SPACE_ID>",
  // idOrName: "<DEPLOYMENT_ID>",
  model: "<MODEL_ID>",
  ...props,
});
```

참고:

- lightweight engine을 사용하지 않는 한 `spaceId`, `projectId` 또는 `idOrName`(deployment id)을 제공해야 합니다. lightweight engine은 이들을 지정하지 않고 작동합니다([watsonx.ai docs](https://www.ibm.com/docs/en/cloud-paks/cp-data/5.0.x?topic=install-choosing-installation-mode) 참조)
- 프로비저닝된 서비스 인스턴스의 지역에 따라 올바른 serviceUrl을 사용하세요.
- model_id를 통해 추론에 사용할 모델을 지정해야 합니다.

### Model Gateway 사용
```typescript
import { WatsonxLLM } from "@langchain/community/llms/ibm";

const props = {
  decoding_method: "sample",
  maxNewTokens: 100,
  minNewTokens: 1,
  temperature: 0.5,
  topK: 50,
  topP: 1,
};

const instance = new WatsonxLLM({
  version: "YYYY-MM-DD",
  serviceUrl: process.env.API_URL,
  model: "<MODEL_ID>",
  modelGateway: true,
  ...props,
});
```

Langchain에서 model gateway를 사용하려면 `@ibm-cloud/watsonx-ai` SDK 또는 `watsonx.ai` API를 통해 provider를 생성하고 모델을 추가해야 합니다. 다음 문서를 참조하세요:
- [API](https://cloud.ibm.com/apidocs/watsonx-ai#create-watsonxai-provider).
- [SDK](https://ibm.github.io/watsonx-ai-node-sdk/modules/1_7_x.gateway.html).

## 호출 및 생성

```javascript
const result = await instance.invoke("Print hello world.");
console.log(result);

const results = await instance.generate([
  "Print hello world.",
  "Print bye, bye world!",
]);
console.log(results);
```

```output

print('Hello world.')<|endoftext|>
{
  generations: [ [ [Object] ], [ [Object] ] ],
  llmOutput: { tokenUsage: { generated_token_count: 28, input_token_count: 10 } }
}
```

## 체이닝

다음과 같이 완성 모델을 prompt template과 체인으로 연결할 수 있습니다:

```javascript
import { PromptTemplate } from "@langchain/core/prompts"

const prompt = PromptTemplate.fromTemplate("How to say {input} in {output_language}:\n")

const chain = prompt.pipe(instance);
await chain.invoke(
  {
    output_language: "German",
    input: "I love programming.",
  }
)
```

```output
Ich liebe Programmieren.

To express that you are passionate about programming in German,
```

## Props 덮어쓰기

초기화 시 전달된 props는 객체의 전체 생명 주기 동안 유지되지만, 아래와 같이 두 번째 인수를 전달하여 단일 메서드 호출에 대해 덮어쓸 수 있습니다

```javascript
const result2 = await instance.invoke("Print hello world.", {
  parameters: {
    maxNewTokens: 100,
  },
});
console.log(result2);
```

```output
print('Hello world.')<|endoftext|>
```

## 토큰화

이 패키지는 사용될 정확한 토큰 수를 반환하는 자체 getNumTokens 구현을 가지고 있습니다.

```javascript
const tokens = await instance.getNumTokens("Print hello world.");
console.log(tokens);
```

```output
4
```
