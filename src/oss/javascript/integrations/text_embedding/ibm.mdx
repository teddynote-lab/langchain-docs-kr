## Setup

IBM WatsonxAI embedding에 액세스하려면 IBM watsonx.ai 계정을 생성하고, API key 또는 다른 유형의 자격 증명을 얻고, `@langchain/community` integration package를 설치해야 합니다.

### Credentials

[IBM Cloud](https://cloud.ibm.com/login)로 이동하여 IBM watsonx.ai에 가입하고 API key를 생성하거나 아래에 제시된 다른 인증 형식을 제공하세요.

#### IAM authentication

```bash
export WATSONX_AI_AUTH_TYPE=iam
export WATSONX_AI_APIKEY=<YOUR-APIKEY>
```

#### Bearer token authentication

```bash
export WATSONX_AI_AUTH_TYPE=bearertoken
export WATSONX_AI_BEARER_TOKEN=<YOUR-BEARER-TOKEN>
```

#### IBM watsonx.ai software authentication

```bash
export WATSONX_AI_AUTH_TYPE=cp4d
export WATSONX_AI_USERNAME=<YOUR_USERNAME>
export WATSONX_AI_PASSWORD=<YOUR_PASSWORD>
export WATSONX_AI_URL=<URL>
```

이러한 값들이 환경 변수에 설정되고 객체가 초기화되면 인증이 자동으로 진행됩니다.

인증은 이러한 값들을 새 인스턴스의 매개변수로 전달하여 수행할 수도 있습니다.

## IAM authentication

```typescript
import { WatsonxEmbeddings } from "@langchain/community/embeddings/ibm";

```

## Instantiation

이제 model 객체를 인스턴스화하고 텍스트를 embed할 수 있습니다:

```javascript
import { WatsonxEmbeddings } from "@langchain/community/embeddings/ibm";

const embeddings = new WatsonxEmbeddings({
  version: "YYYY-MM-DD",
  serviceUrl: process.env.API_URL,
  projectId: "<PROJECT_ID>",
  spaceId: "<SPACE_ID>",
  model: "<MODEL_ID>",
});
```

참고:

- 진행하려면 `spaceId` 또는 `projectId`를 제공해야 합니다.
- 프로비저닝된 서비스 인스턴스의 지역에 따라 올바른 serviceUrl을 사용하세요.

### Model Gateway
```typescript
import { WatsonxEmbeddings } from "@langchain/community/embeddings/ibm";

const instance = new WatsonxEmbeddings({
  version: "YYYY-MM-DD",
  serviceUrl: process.env.API_URL,
  model: "<ALIAS_MODEL_ID>",
  modelGateway: true,
});
```

## Indexing and Retrieval

Embedding model은 데이터 인덱싱과 나중에 검색하는 과정 모두에서 retrieval-augmented generation (RAG) 플로우에 자주 사용됩니다. 더 자세한 지침은 [**Learn** 탭](/oss/learn/) 아래의 RAG 튜토리얼을 참조하세요.

아래에서는 위에서 초기화한 `embeddings` 객체를 사용하여 데이터를 인덱싱하고 검색하는 방법을 확인할 수 있습니다. 이 예제에서는 데모 [`MemoryVectorStore`](/oss/integrations/vectorstores/memory)를 사용하여 샘플 문서를 인덱싱하고 검색합니다.

```javascript
// Create a vector store with a sample text
import { MemoryVectorStore } from "@langchain/classic/vectorstores/memory";

const text = "LangChain is the framework for building context-aware reasoning applications";

const vectorstore = await MemoryVectorStore.fromDocuments(
  [{ pageContent: text, metadata: {} }],
  embeddings,
);

// Use the vector store as a retriever that returns a single document
const retriever = vectorstore.asRetriever(1);

// Retrieve the most similar text
const retrievedDocuments = await retriever.invoke("What is LangChain?");

retrievedDocuments[0].pageContent;
```

```output
LangChain is the framework for building context-aware reasoning applications
```

## Direct Usage

내부적으로 vectorstore와 retriever 구현은 `embeddings.embedDocument(...)`와 `embeddings.embedQuery(...)`를 호출하여 `fromDocuments`와 retriever의 `invoke` 작업에서 각각 사용되는 텍스트에 대한 embedding을 생성합니다.

이러한 메서드를 직접 호출하여 자신의 사용 사례에 맞는 embedding을 얻을 수 있습니다.

### Embed single texts

검색을 위해 `embedQuery`로 쿼리를 embed할 수 있습니다. 이는 쿼리에 특화된 벡터 표현을 생성합니다:

```javascript
    const singleVector = await embeddings.embedQuery(text);
    singleVector.slice(0, 10);
```

```output
[
   -0.017436018,  -0.01469498,
   -0.015685871, -0.013543149,
  -0.0011519607, -0.008123747,
    0.015286108, -0.023845721,
    -0.02454774,   0.07235078
]
```

### Embed multiple texts

인덱싱을 위해 `embedDocuments`로 여러 텍스트를 embed할 수 있습니다. 이 메서드에 사용되는 내부 구현은 쿼리 embedding과 다를 수 있습니다(반드시 다를 필요는 없습니다):

```javascript
    const text2 = "LangGraph is a library for building stateful, multi-actor applications with LLMs";

    const vectors = await embeddings.embedDocuments([text, text2]);

    console.log(vectors[0].slice(0, 10));
    console.log(vectors[1].slice(0, 10));

```
