    ## API 안정성

    LangChain 1.0의 시맨틱 버저닝 채택과 함께:
    - public API의 호환성을 깨는 변경사항은 메이저 버전 릴리스에서만 발생합니다 (예: `2.0.0`)
    - 마이너 버전 업데이트 (예: `1.0.0`에서 `1.1.0`으로)는 호환성을 깨지 않고 새로운 기능을 추가합니다
    - 패치 버전 업데이트 (예: `1.0.0`에서 `1.0.1`로)는 버그 수정과 사소한 개선사항을 포함합니다

    우리는 일반적으로 불필요한 변경을 피하려고 노력하며, 제거되는 기능에 대한 지원 중단 정책을 제공할 것입니다.

    ### 다른 패키지의 안정성

    LangChain 생태계의 다른 패키지들의 안정성은 다를 수 있습니다:

    - **LangChain이 유지관리하는 파트너 패키지** (`langchain-openai`와 `langchain-anthropic` 같은)는 시맨틱 버저닝을 따르며 1.0 이후 안정적일 것으로 예상됩니다. 다른 파트너 패키지들은 다른 안정성 및 버저닝 정책을 따를 수 있으며, 사용자는 해당 패키지의 문서를 참조해야 합니다.

    :::python
    - **`langchain-community`**는 서드파티 통합을 포함하는 커뮤니티 유지관리 패키지입니다. 통합의 수가 많기 때문에, `langchain-community`는 `langchain`과 `langchain-core`와 동일한 엄격한 시맨틱 버저닝 정책을 따르지 않습니다. 자세한 내용은 장기 지원의 "특별 고려사항" 섹션을 참조하세요.
    :::
    :::js
    - **`@langchain/community`**는 서드파티 통합을 포함하는 커뮤니티 유지관리 패키지입니다. 커뮤니티 기여와 서드파티 의존성의 특성상, `@langchain/community`는 `langchain`과 `@langchain/core`와 동일한 엄격한 시맨틱 버저닝 정책을 따르지 않습니다. 자세한 내용은 장기 지원의 "특별 고려사항" 섹션을 참조하세요.
    :::

    :::python

    - 권장되는 대안으로의 마이그레이션 가이드를 제공합니다
    - 모든 1.x 마이너 릴리스를 통해 지원 중단된 기능에 대한 보안 업데이트를 제공합니다

    ### 릴리스 상태 정의

    패키지는 다음 상태 중 하나로 표시됩니다:
    - **ACTIVE**: 현재 활발한 개발 중, 버그 수정, 보안 패치 및 새로운 기능 포함

    ### 현재 LTS 릴리스

    - **상태**: 2.0 릴리스까지 ACTIVE
    - **지원 기간**: 2.0이 릴리스된 후, 1.0은 최소 1년 동안 MAINTENANCE 모드로 전환됩니다
    - **Semver 준수**: 사용자는 호환성을 깨지 않고 마이너 버전 간 업그레이드할 수 있습니다 (예: 1.0에서 1.1로)

    ## API 안정성

    ### 안정적인 API

    특별한 접두사가 없는 모든 API는 안정적이며 프로덕션 사용 준비가 된 것으로 간주됩니다. 우리는 메이저 버전 내에서 안정적인 기능에 대한 하위 호환성을 유지합니다.

    ### 베타 기능

    문서에서 `beta`로 표시된 기능은:

    - 기능이 완성되고 테스트되었습니다
    - 변경될 수 있다는 이해와 함께 프로덕션 사용이 안전합니다
    - 사용자 피드백을 기반으로 사소한 API 조정이 있을 수 있습니다

    ### 실험적 기능

    `experimental` 또는 `alpha`로 표시된 기능:

    - 활발히 개발 중입니다
    - 크게 변경되거나 제거될 수 있습니다
    - 프로덕션에서 주의해서 사용해야 합니다

    ### 내부 API

    밑줄(`_`)로 시작하거나 명시적으로 내부로 표시된 API:

    - public API의 일부가 아닙니다
    - 예고 없이 변경될 수 있습니다
    - 직접 사용해서는 안 됩니다

    ## 지원 중단 정책

    기능을 지원 중단할 때:

    1. **지원 중단 공지**: 기능은 명확한 마이그레이션 가이드와 함께 지원 중단으로 표시됩니다
    2. **유예 기간**: 지원 중단된 기능은 최소 하나의 마이너 버전 동안 계속 작동합니다
    3. **제거**: 기능은 메이저 버전 릴리스에서만 제거됩니다
    4. **마이그레이션 지원**: 우리는 마이그레이션 가이드를 제공하며, 가능한 경우 자동화 도구를 제공합니다

    ## 호환성을 깨는 변경사항

    호환성을 깨는 변경사항은 메이저 버전에서만 도입되며 다음을 포함합니다:
    - 지원 중단된 API의 제거
    - 필수 매개변수의 변경
    - 기존 애플리케이션에 영향을 미치는 기본 동작의 변경
    - 최소 Python/Node.js 버전 업데이트

    ## 마이그레이션 지원

    메이저 버전 업그레이드를 위해 우리는 다음을 제공합니다:
    - 포괄적인 마이그레이션 가이드
    - 가능한 경우 자동화된 마이그레이션 스크립트
    - 이전 메이저 버전에 대한 연장된 지원 기간
    - 모든 호환성을 깨는 변경사항에 대한 명확한 문서
