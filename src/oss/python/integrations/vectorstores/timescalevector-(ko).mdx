>[Timescale Vector](https://www.timescale.com/ai?utm_campaign=vectorlaunch&utm_source=langchain&utm_medium=referral)는 AI 애플리케이션을 위한 `PostgreSQL++` vector database입니다.

이 노트북은 Postgres vector database인 `Timescale Vector`를 사용하는 방법을 보여줍니다. 이 예제에서는 TimescaleVector를 사용하여 (1) semantic search, (2) 시간 기반 vector search, (3) self-querying, (4) 쿼리 속도를 높이기 위한 인덱스 생성 방법을 학습합니다.

## Timescale Vector란?

`Timescale Vector`는 `PostgreSQL`에서 수백만 개의 vector embeddings를 효율적으로 저장하고 쿼리할 수 있게 합니다.

- `pgvector`를 강화하여 `DiskANN`에서 영감을 받은 인덱싱 알고리즘으로 1억+ 벡터에서 더 빠르고 정확한 similarity search를 제공합니다.
- 자동 시간 기반 파티셔닝과 인덱싱을 통해 빠른 시간 기반 vector search를 가능하게 합니다.
- vector embeddings와 관계형 데이터를 쿼리하기 위한 친숙한 SQL 인터페이스를 제공합니다.

`Timescale Vector`는 POC부터 프로덕션까지 확장 가능한 클라우드 `PostgreSQL` 기반 AI 데이터베이스입니다:

- 관계형 메타데이터, vector embeddings, 시계열 데이터를 하나의 데이터베이스에 저장하여 운영을 단순화합니다.
- 스트리밍 백업과 복제, 고가용성, row-level security 등 엔터프라이즈급 기능을 갖춘 견고한 PostgreSQL 기반의 이점을 제공합니다.
- 엔터프라이즈급 보안과 컴플라이언스로 걱정 없는 사용 경험을 제공합니다.

## 2. 시간 기반 필터링을 사용한 Similarity Search

Timescale Vector의 핵심 사용 사례 중 하나는 효율적인 시간 기반 vector search입니다. Timescale Vector는 벡터(및 관련 메타데이터)를 자동으로 시간 기준으로 파티셔닝함으로써 이를 가능하게 합니다. 이를 통해 질의 벡터와의 유사도뿐 아니라 시간 기준으로도 벡터를 효율적으로 쿼리할 수 있습니다.

시간 기반 vector search 기능은 다음과 같은 애플리케이션에 유용합니다:

- LLM 응답 히스토리 저장 및 검색(예: 챗봇)
- 질의 벡터와 유사한 최신 embeddings 찾기(예: 최신 뉴스)
- 관련 시간 범위로 similarity search를 제한하기(예: 지식 베이스에 대한 시간 기반 질의)

TimescaleVector의 시간 기반 vector search 사용법을 보여주기 위해, TimescaleDB의 git log 히스토리에 대해 질문을 해보겠습니다. 시간 기반 uuid와 함께 문서를 추가하는 방법과 시간 범위 필터를 사용하여 similarity search를 실행하는 방법을 설명합니다.

타임스탬프를 기반으로 문서와 관련 vector embedding을 위한 uuid를 생성하는 helper function을 정의합니다. 이 함수를 사용하여 각 git log 항목에 대한 uuid를 생성합니다.

중요 참고: 문서를 다루면서 시간 기반 검색을 위해 벡터에 현재 날짜와 시간을 연관시키고 싶다면, 이 단계를 건너뛸 수 있습니다. 문서가 적재될 때 uuid는 기본적으로 자동 생성됩니다.

```python
from timescale_vector import client


# Function to take in a date string in the past and return a uuid v1
def create_uuid(date_string: str):
    if date_string is None:
        return None
    time_format = "%a %b %d %H:%M:%S %Y %z"
    datetime_obj = datetime.strptime(date_string, time_format)
    uuid = client.uuid_from_time(datetime_obj)
    return str(uuid)
```

다음으로 JSON 레코드에서 관련 메타데이터를 추출하는 metadata function을 정의합니다. 이 함수를 JSONLoader에 전달합니다. 자세한 내용은 [JSON document loader 문서](/oss/integrations/document_loaders/json)를 참고하세요.

```python
# Helper function to split name and email given an author string consisting of Name Lastname <email>
def split_name(input_string: str) -> Tuple[str, str]:
    if input_string is None:
        return None, None
    start = input_string.find("<")
    end = input_string.find(">")
    name = input_string[:start].strip()
    email = input_string[start + 1 : end].strip()
    return name, email


# Helper function to transform a date string into a timestamp_tz string
def create_date(input_string: str) -> datetime:
    if input_string is None:
        return None
    # Define a dictionary to map month abbreviations to their numerical equivalents
    month_dict = {
        "Jan": "01",
        "Feb": "02",
        "Mar": "03",
        "Apr": "04",
        "May": "05",
        "Jun": "06",
        "Jul": "07",
        "Aug": "08",
        "Sep": "09",
        "Oct": "10",
        "Nov": "11",
        "Dec": "12",
    }

    # Split the input string into its components
    components = input_string.split()
    # Extract relevant information
    day = components[2]
    month = month_dict[components[1]]
    year = components[4]
    time = components[3]
    timezone_offset_minutes = int(components[5])  # Convert the offset to minutes
    timezone_hours = timezone_offset_minutes // 60  # Calculate the hours
    timezone_minutes = timezone_offset_minutes % 60  # Calculate the remaining minutes
    # Create a formatted string for the timestamptz in PostgreSQL format
    timestamp_tz_str = (
        f"{year}-{month}-{day} {time}+{timezone_hours:02}{timezone_minutes:02}"
    )
    return timestamp_tz_str


# Metadata extraction function to extract metadata from a JSON record
def extract_metadata(record: dict, metadata: dict) -> dict:
    record_name, record_email = split_name(record["author"])
    metadata["id"] = create_uuid(record["date"])
    metadata["date"] = create_date(record["date"])
    metadata["author_name"] = record_name
    metadata["author_email"] = record_email
    metadata["commit_hash"] = record["commit"]
    return metadata
```

이제 전처리를 마친 문서 컬렉션에서 Timescale Vector 인스턴스를 생성합니다.

먼저 PostgreSQL 데이터베이스에서 테이블 이름이 될 collection name을 정의합니다.

또한 `time_partition_interval` 인자로 전달할 time delta를 정의합니다. 이는 데이터를 시간 기준으로 파티셔닝할 때 사용하는 간격입니다. 각 파티션은 지정된 기간의 데이터로 구성됩니다. 여기서는 단순화를 위해 7일을 사용하지만, 사용 사례에 맞는 값을 선택할 수 있습니다. 예를 들어 최근 벡터를 자주 조회한다면 1일 같은 더 작은 time delta를, 수년 단위의 긴 기간을 조회한다면 6개월 또는 1년 같은 더 큰 time delta를 사용할 수 있습니다.

마지막으로 TimescaleVector 인스턴스를 생성합니다. 위 전처리 단계에서 메타데이터에 만든 `uuid` 필드를 `ids` 인자로 지정합니다. 이렇게 하는 이유는 uuid의 시간 부분이 과거 날짜(예: 커밋이 이루어진 시점)를 반영하길 원하기 때문입니다. 반대로 문서에 현재 날짜와 시간을 연관시키고 싶다면 id 인자를 제거하면 uuid가 현재 날짜와 시간으로 자동 생성됩니다.

```python
# Define collection name
COLLECTION_NAME = "timescale_commits"
embeddings = OpenAIEmbeddings()

# Create a Timescale Vector instance from the collection of documents
db = TimescaleVector.from_documents(
    embedding=embeddings,
    ids=[doc.metadata["id"] for doc in docs],
    documents=docs,
    collection_name=COLLECTION_NAME,
    service_url=SERVICE_URL,
    time_partition_interval=timedelta(days=7),
)
```

## 3. ANN Search Index를 사용해 쿼리 가속화

embedding 컬럼에 인덱스를 생성하면 similarity 쿼리를 가속화할 수 있습니다. 이는 데이터의 상당 부분을 적재한 후에 수행하는 것이 좋습니다.

Timescale Vector는 다음 인덱스를 지원합니다:

- timescale_vector index (tsv): 빠른 similarity search를 위한 disk-ann에서 영감을 받은 그래프 인덱스(기본값).
- pgvector의 HNSW 인덱스: 빠른 similarity search를 위한 hierarchical navigable small world 그래프 인덱스.
- pgvector의 IVFFLAT 인덱스: 빠른 similarity search를 위한 inverted file 인덱스.

중요 참고: PostgreSQL에서 각 테이블은 특정 컬럼에 대해 하나의 인덱스만 가질 수 있습니다. 따라서 서로 다른 인덱스 유형의 성능을 테스트하려면 (1) 서로 다른 인덱스를 가진 여러 테이블을 만들거나, (2) 동일한 테이블에 여러 vector 컬럼을 만들고 각 컬럼에 서로 다른 인덱스를 생성하거나, (3) 동일 컬럼의 인덱스를 삭제 후 다른 유형으로 재생성하여 결과를 비교할 수 있습니다.

```python
# Initialize an existing TimescaleVector store
```

## 4. Timescale Vector로 Self Querying Retriever 사용하기

Timescale Vector는 self-querying retriever 기능도 지원하여, 스스로를 쿼리할 수 있는 능력을 제공합니다. 질의문과 필터(단일 또는 합성)가 포함된 자연어 질의가 주어지면, retriever는 쿼리를 구성하는 LLM 체인을 사용해 SQL 쿼리를 작성한 뒤, Timescale Vector vectorstore의 기반 PostgreSQL 데이터베이스에 이를 적용합니다.

Timescale Vector에서 self-querying을 설명하기 위해, 3부에서 사용한 동일한 gitlog 데이터셋을 사용하겠습니다.

```python
COLLECTION_NAME = "timescale_commits"
vectorstore = TimescaleVector(
    embedding_function=OpenAIEmbeddings(),
    collection_name=COLLECTION_NAME,
    service_url=SERVICE_URL,
)
```

다음으로 self-querying retriever를 생성합니다. 이를 위해 문서가 지원하는 메타데이터 필드 정보와 문서 내용에 대한 간단한 설명을 사전에 제공해야 합니다.

```python
from langchain.chains.query_constructor.base import AttributeInfo
from langchain.retrievers.self_query.base import SelfQueryRetriever
from langchain_openai import OpenAI

# Give LLM info about the metadata fields
metadata_field_info = [
    AttributeInfo(
        name="id",
        description="A UUID v1 generated from the date of the commit",
        type="uuid",
    ),
    AttributeInfo(
        name="date",
        description="The date of the commit in timestamptz format",
        type="timestamptz",
    ),
    AttributeInfo(
        name="author_name",
        description="The name of the author of the commit",
        type="string",
    ),
    AttributeInfo(
        name="author_email",
        description="The email address of the author of the commit",
        type="string",
    ),
]
document_content_description = "The git log commit summary containing the commit hash, author, date of commit, change summary and change details"

# Instantiate the self-query retriever from an LLM
llm = OpenAI(temperature=0)
retriever = SelfQueryRetriever.from_llm(
    llm,
    vectorstore,
    document_content_description,
    metadata_field_info,
    enable_limit=True,
    verbose=True,
)
```

이제 gitlog 데이터셋에서 self-querying retriever를 테스트해봅시다.

아래 쿼리를 실행해 보면, 자연어로 질의, 필터와 함께하는 질의, 합성 필터(AND, OR가 있는 필터)를 포함한 질의를 지정하면 self-query retriever가 해당 질의를 SQL로 변환해 Timescale Vector PostgreSQL vectorstore에서 검색을 수행하는 것을 확인할 수 있습니다.

## 5. 기존 TimescaleVector vectorstore와 작업하기

위 예제들에서는 문서 컬렉션에서 vectorstore를 생성했습니다. 그러나 기존 vectorstore에 데이터를 삽입하고 쿼리해야 하는 경우가 자주 있습니다. TimescaleVector vector store에서 기존 문서 컬렉션을 초기화하고, 문서를 추가하고, 쿼리하는 방법을 살펴보겠습니다.

기존 Timescale Vector store로 작업하려면, 쿼리할 테이블의 이름(`COLLECTION_NAME`)과 클라우드 PostgreSQL 데이터베이스의 URL(`SERVICE_URL`)을 알아야 합니다.

```python
# Initialize the existing
COLLECTION_NAME = "timescale_commits"
embeddings = OpenAIEmbeddings()
vectorstore = TimescaleVector(
        collection_name=COLLECTION_NAME,
        service_url=SERVICE_URL,
        embedding_function=embeddings,
)
```

테이블에 새 데이터를 로드하려면 `add_document()` 함수를 사용합니다. 이 함수는 문서 목록과 메타데이터 목록을 받습니다. 메타데이터에는 각 문서에 대한 고유 id가 포함되어야 합니다.

문서를 현재 날짜와 시간과 연관시키고 싶다면 id 목록을 생성할 필요가 없습니다. 문서마다 uuid가 자동으로 생성됩니다.

문서를 과거의 특정 날짜와 시간과 연관시키고 싶다면, 위 2절에서 보여준 것처럼 `timecale-vector` Python 라이브러리의 `uuid_from_time` 함수를 사용해 id 목록을 만들 수 있습니다. 이 함수는 datetime 객체를 받아 날짜와 시간이 인코딩된 uuid를 반환합니다.

```python
# Add documents to a collection in TimescaleVector
ids = vectorstore.add_documents([Document(page_content="foo")])
ids
```

### 데이터 삭제

uuid 또는 메타데이터 필터로 데이터를 삭제할 수 있습니다.

```python
ids = vectorstore.add_documents([Document(page_content="Bar")])

vectorstore.delete(ids)
```

```output
True
```

메타데이터를 사용한 삭제는 특정 소스에서 주기적으로 스크래핑한 정보, 특정 날짜, 기타 메타데이터 속성에 기반해 정보를 주기적으로 갱신하고자 할 때 특히 유용합니다.

```python
vectorstore.add_documents(
        [Document(page_content="Hello World", metadata={"source": "www.example.com/hello"})]
)
vectorstore.add_documents(
        [Document(page_content="Adios", metadata={"source": "www.example.com/adios"})]
)

vectorstore.delete_by_metadata({"source": "www.example.com/adios"})

vectorstore.add_documents(
    [
        Document(
                        page_content="Adios, but newer!",
                        metadata={"source": "www.example.com/adios"},
        )
    ]
)
```

### vectorstore 덮어쓰기

기존 collection이 있다면, `from_documents`를 수행하고 `pre_delete_collection` = True로 설정하여 덮어쓸 수 있습니다.

```python
db = TimescaleVector.from_documents(
        documents=docs,
        embedding=embeddings,
        collection_name=COLLECTION_NAME,
        service_url=SERVICE_URL,
        pre_delete_collection=True,
)
```

```python
docs_with_score = db.similarity_search_with_score("foo")
```
