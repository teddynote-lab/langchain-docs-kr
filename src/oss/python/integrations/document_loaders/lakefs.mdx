>[lakeFS](https://docs.lakefs.io/)는 data lake에 대한 확장 가능한 버전 관리를 제공하며, Git과 유사한 의미론을 사용하여 해당 버전을 생성하고 액세스합니다.

이 notebook은 `lakeFS` 경로(객체 또는 prefix)에서 document 객체를 로드하는 방법을 다룹니다.

## lakeFS loader 초기화

`ENDPOINT`, `LAKEFS_ACCESS_KEY`, `LAKEFS_SECRET_KEY` 값을 자신의 값으로 교체하세요.

```python
from langchain_community.document_loaders import LakeFSLoader
```

```python
ENDPOINT = ""
LAKEFS_ACCESS_KEY = ""
LAKEFS_SECRET_KEY = ""

lakefs_loader = LakeFSLoader(
    lakefs_access_key=LAKEFS_ACCESS_KEY,
    lakefs_secret_key=LAKEFS_SECRET_KEY,
    lakefs_endpoint=ENDPOINT,
)
```

## 경로 지정

로드할 파일을 제어하기 위해 prefix 또는 완전한 객체 경로를 지정할 수 있습니다.

document를 로드할 repository, reference (branch, commit id, 또는 tag), 그리고 경로를 해당하는 `REPO`, `REF`, `PATH`에 지정하세요:

```python
REPO = ""
REF = ""
PATH = ""

lakefs_loader.set_repo(REPO)
lakefs_loader.set_ref(REF)
lakefs_loader.set_path(PATH)

docs = lakefs_loader.load()
docs
```
