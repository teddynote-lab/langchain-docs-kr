## Retriever 예제

이 노트북은 [Zep Long-term Memory Store](https://getzep.github.io/)를 사용하여 과거 채팅 메시지 기록을 검색하는 방법을 보여줍니다.

다음을 시연합니다:

1. Zep memory store에 대화 기록 추가하기.
2. 대화 기록에 대한 vector 검색:
    1. 채팅 메시지에 대한 유사도 검색
    2. 채팅 메시지 검색의 maximal marginal relevance 재순위 사용
    3. metadata filter를 사용한 검색 필터링
    4. 채팅 메시지 요약에 대한 유사도 검색
    5. 요약 검색의 maximal marginal relevance 재순위 사용

```python
import getpass
import time
from uuid import uuid4

from langchain_community.memory.zep_memory import ZepMemory
from langchain.messages import AIMessage, HumanMessage

# Set this to your Zep server URL
ZEP_API_URL = "http://localhost:8000"
```

### Zep Chat Message History Class 초기화 및 memory store에 채팅 메시지 기록 추가

**참고:** 다른 Retriever와 달리 Zep Retriever가 반환하는 콘텐츠는 session/user별로 다릅니다. Retriever를 인스턴스화할 때 `session_id`가 필요합니다.

```python
# Provide your Zep API key. Note that this is optional. See https://docs.getzep.com/deployment/auth
AUTHENTICATE = False

zep_api_key = None
if AUTHENTICATE:
    zep_api_key = getpass.getpass()
```

```python
session_id = str(uuid4())  # This is a unique identifier for the user/session

# Initialize the Zep Memory Class
zep_memory = ZepMemory(session_id=session_id, url=ZEP_API_URL, api_key=zep_api_key)
```

### Zep Retriever를 사용하여 Zep memory에 대한 vector 검색

Zep은 과거 대화 메모리에 대한 네이티브 vector 검색을 제공합니다. Embedding은 자동으로 발생합니다.

참고: 메시지의 embedding은 비동기적으로 발생하므로 첫 번째 쿼리는 결과를 반환하지 않을 수 있습니다. embedding이 생성되면 후속 쿼리가 결과를 반환합니다.

```python
from langchain_community.retrievers.zep import SearchScope, SearchType, ZepRetriever

zep_retriever = ZepRetriever(
    session_id=session_id,  # Ensure that you provide the session_id when instantiating the Retriever
    url=ZEP_API_URL,
    top_k=5,
    api_key=zep_api_key,
)

await zep_retriever.ainvoke("Who wrote Parable of the Sower?")
```

### MMR(Maximal Marginal Relevance)을 사용한 재순위

Zep은 MMR을 사용한 결과 재순위를 위한 네이티브 SIMD 가속 지원을 제공합니다. 이는 결과의 중복을 제거하는 데 유용합니다.

```python
zep_retriever = ZepRetriever(
    session_id=session_id,  # Ensure that you provide the session_id when instantiating the Retriever
    url=ZEP_API_URL,
    top_k=5,
    api_key=zep_api_key,
    search_type=SearchType.mmr,
    mmr_lambda=0.5,
)

await zep_retriever.ainvoke("Who wrote Parable of the Sower?")
```

### metadata filter를 사용하여 검색 결과 정제

Zep은 metadata로 결과를 필터링하는 것을 지원합니다. 이는 entity type이나 기타 metadata로 결과를 필터링하는 데 유용합니다.

자세한 정보: [docs.getzep.com/sdk/search_query/](https://docs.getzep.com/sdk/search_query/)

```python
filter = {"where": {"jsonpath": '$[*] ? (@.Label == "WORK_OF_ART")'}}

await zep_retriever.ainvoke("Who wrote Parable of the Sower?", metadata=filter)
```

### MMR 재순위를 사용한 요약 검색

Zep은 채팅 메시지의 요약을 자동으로 생성합니다. 이러한 요약은 Zep Retriever를 사용하여 검색할 수 있습니다. 요약은 대화의 정제된 내용이므로 검색 쿼리와 일치할 가능성이 더 높고 LLM에 풍부하고 간결한 컨텍스트를 제공합니다.

연속적인 요약에는 유사한 콘텐츠가 포함될 수 있으며, Zep의 유사도 검색은 가장 일치하는 결과를 반환하지만 다양성은 거의 없습니다.
MMR은 prompt에 채워지는 요약이 관련성이 있으면서 각각 LLM에 추가 정보를 제공하도록 결과를 재순위합니다.

```python
zep_retriever = ZepRetriever(
    session_id=session_id,  # Ensure that you provide the session_id when instantiating the Retriever
    url=ZEP_API_URL,
    top_k=3,
    api_key=zep_api_key,
    search_scope=SearchScope.summary,
    search_type=SearchType.mmr,
    mmr_lambda=0.5,
)

await zep_retriever.ainvoke("Who wrote Parable of the Sower?")
```
