**환영합니다! 기여에 관심을 가져주셔서 감사합니다.**

LangChain은 생성형 AI 분야에서 가장 큰 개발자 커뮤니티를 형성하는 데 도움을 주었으며, 우리는 항상 새로운 기여자를 환영합니다. 버그 수정, 기능 추가, 문서 개선, 피드백 공유 등 어떤 방식이든 여러분의 참여는 LangChain과 LangGraph를 모두를 위해 더 나아지게 만듭니다.

## 기여 방법

<AccordionGroup>
    <Accordion title="버그 신고" icon="bug">
        버그를 발견하셨나요? 다음 단계를 따라 수정에 도움을 주세요:

        <Steps>
            <Step title="검색">
                해당 저장소의 GitHub Issues에서 이미 존재하는 이슈인지 확인하세요:

                :::python
                <Columns cols={2}>
                    <Card title="LangChain" icon="link" href="https://github.com/langchain-ai/langchain/issues">Issues</Card>
                    <Card title="LangGraph" icon="circle-nodes" href="https://github.com/langchain-ai/langgraph/issues">Issues</Card>
                </Columns>
                :::
                :::js
                <Columns cols={2}>
                    <Card title="LangChain" icon="link" href="https://github.com/langchain-ai/langchainjs/issues">Issues</Card>
                    <Card title="LangGraph" icon="circle-nodes" href="https://github.com/langchain-ai/langgraphjs/issues">Issues</Card>
                </Columns>
                :::
            </Step>
            <Step title="이슈 생성">
                이슈가 없다면 새로 생성하세요. 작성 시 제공된 템플릿을 따르고 [최소한의 재현 가능한 예제](https://stackoverflow.com/help/minimal-reproducible-example)를 포함해야 합니다. 생성된 이슈에 관련 레이블을 첨부하세요. 프로젝트 관리자가 이슈를 재현할 수 없는 경우, 적시에 처리되지 않을 가능성이 높습니다.
            </Step>
            <Step title="대기">
                프로젝트 관리자가 이슈를 분류하고 추가 정보를 요청할 수 있습니다. 많은 양의 이슈를 관리하고 있으니 인내심을 가져주세요. 새로운 정보를 제공할 것이 없다면 이슈를 재촉하지 마세요.
            </Step>
        </Steps>

        이슈를 추가할 때는 단일 주제에 집중하도록 노력하세요. 두 이슈가 관련되어 있거나 차단 관계에 있다면, 결합하기보다는 [링크](https://docs.github.com/en/issues/tracking-your-work-with-issues/using-issues/linking-a-pull-request-to-an-issue#linking-a-pull-request-to-an-issue-using-a-keyword)로 연결하세요. 예를 들어,

        ```text
        This issue is blocked by #123 and related to #456.
        ```
    </Accordion>

    <Accordion title="기능 제안" icon="wand-magic-sparkles">
        새로운 기능이나 개선 사항에 대한 아이디어가 있으신가요?

        <Steps>
            <Step title="검색">
                해당 저장소의 이슈에서 기존 기능 요청을 검색하세요:

                :::python
                <Columns cols={2}>
                    <Card title="LangChain" icon="link" href="https://github.com/langchain-ai/langchain/issues?q=state%3Aopen%20label%3A%22feature%20request%22">Issues</Card>
                    <Card title="LangGraph" icon="circle-nodes" href="https://github.com/langchain-ai/langgraph/issues?q=state%3Aopen%20label%3Aenhancement">Issues</Card>
                </Columns>
                :::
                :::js
                <Columns cols={2}>
                    <Card title="LangChain" icon="link" href="https://github.com/langchain-ai/langchainjs">Issues</Card>
                    <Card title="LangGraph" icon="circle-nodes" href="https://github.com/langchain-ai/langgraphjs/labels?q=feature">Issues</Card>
                </Columns>
                :::
            </Step>
            <Step title="논의">
                :::python
                요청이 없다면 [관련 카테고리](https://forum.langchain.com/c/help/langchain/14)에서 새로운 논의를 시작하여 프로젝트 관리자와 커뮤니티가 피드백을 제공할 수 있도록 하세요.
                :::
                :::js
                요청이 없다면 [관련 카테고리](https://forum.langchain.com/c/help/langchain/13)에서 새로운 논의를 시작하여 프로젝트 관리자와 커뮤니티가 피드백을 제공할 수 있도록 하세요.
                :::
            </Step>
            <Step title="설명">
                사용 사례와 다른 사람들에게 왜 가치가 있는지 설명하세요. 가능하다면 해당되는 경우 예제나 모형을 제공하세요. 통과해야 하는 테스트 케이스를 개요로 작성하세요.
            </Step>
        </Steps>
    </Accordion>

    <Accordion title="문서 개선" icon="book">
