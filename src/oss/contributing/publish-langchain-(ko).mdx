**커뮤니티에서 사용할 수 있도록 통합 패키지를 배포하세요.**

:::python
<Warning>
**중요: 새로운 통합 패키지는 LangChain monorepo에 대한 PR이 아닌 독립 실행형 패키지여야 합니다.**

LangChain은 메인 저장소에서 일부 자사 및 높은 사용량의 통합 패키지(OpenAI, Anthropic, Ollama 등)를 유지 관리하지만, **새로운 통합 패키지는 별도의 PyPI 패키지 및 저장소로 배포되어야 합니다**(예: `langchain-yourservice`). 사용자는 핵심 LangChain 패키지와 함께 이를 설치합니다. 메인 LangChain 저장소에 통합 패키지를 직접 추가하는 PR을 제출해서는 **안 됩니다**.
</Warning>

이제 패키지가 구현되고 테스트되었으므로, 이를 배포하고 문서를 추가하여 커뮤니티에서 발견할 수 있도록 할 수 있습니다.

## 패키지 배포하기

<Info>
    이 가이드는 이미 패키지를 구현하고 테스트를 작성했다고 가정합니다. 아직 작성하지 않았다면 [구현 가이드](/oss/contributing/implement-langchain)와 [테스트 가이드](/oss/contributing/standard-tests-langchain)를 참조하세요.
</Info>

이 가이드에서는 패키지 레지스트리로 PyPI를 사용합니다. 원하는 경우 다른 레지스트리에 배포할 수도 있으며, 지침은 다를 수 있습니다.

### 자격 증명 설정

먼저 PyPI 계정이 있는지 확인하세요:

<AccordionGroup>
    <Accordion title="PyPI Token 생성 방법" icon="key">
        <Steps>
            <Step title="계정 생성">
                [PyPI 웹사이트](https://pypi.org/)로 이동하여 계정을 생성하세요
            </Step>

            <Step title="이메일 인증">
                PyPI에서 보낸 링크를 클릭하여 이메일 주소를 인증하세요
            </Step>

            <Step title="2FA 활성화">
                계정 설정으로 이동하여 "Generate Recovery Codes"를 클릭하여 2FA를 활성화하세요. API token을 생성하려면 2FA를 **반드시** 활성화해야 합니다
            </Step>

            <Step title="토큰 생성">
                계정 설정으로 이동하여 [새 API token을 생성](https://pypi.org/manage/account/token/)하세요
            </Step>
        </Steps>
    </Accordion>
</AccordionGroup>

## 문서 추가하기

[통합 탭](/oss/integrations/providers/overview) 아래 이 사이트에 패키지에 대한 문서를 추가하려면 관련 문서 페이지를 생성하고 [LangChain docs 저장소](https://github.com/langchain-ai/docs)에 PR을 열어야 합니다.

### 문서 작성하기

구축한 통합 패키지의 유형에 따라 다양한 유형의 문서 페이지를 생성해야 합니다. LangChain은 시작하는 데 도움이 되도록 다양한 유형의 통합 패키지에 대한 템플릿을 제공합니다.

    특정 문서 페이지를 원시 markdown으로 보려면 페이지 오른쪽 상단의 "Copy page" 옆에 있는 드롭다운 버튼을 사용하고 "View as Markdown"을 선택하세요.
</Tip>

### PR 제출하기

개인 GitHub 계정으로 [LangChain docs 저장소](https://github.com/langchain-ai/docs)를 fork하고 로컬에 clone하세요. 통합 패키지를 위한 새 브랜치를 생성하세요. 템플릿을 복사하고 선호하는 markdown 텍스트 편집기를 사용하여 수정하세요. 문서를 작성할 때 [문서 가이드](/oss/contributing/documentation)를 참조하고 따르세요.

<Warning>
    다음과 같은 경우 PR을 거부하거나 수정을 요청할 수 있습니다:
    - CI 검사가 실패하는 경우
    - 심각한 문법 오류나 오타가 있는 경우
    - [Mintlify components](/oss/contributing/documentation#mintlify-components)가 잘못 사용된 경우
    - 페이지에 [frontmatter](/oss/contributing/documentation#page-structure)가 누락된 경우
    - [현지화](/oss/contributing/documentation#localization)가 누락된 경우(해당되는 경우)
    - [코드 예제](/oss/contributing/documentation#in-code-documentation)가 실행되지 않거나 오류가 있는 경우
    - [품질 기준](/oss/contributing/documentation#quality-standards)을 충족하지 않는 경우
</Warning>

많은 양의 PR을 처리하므로 인내심을 가져주세요. 가능한 한 빨리 PR을 검토하고 피드백을 제공하거나 병합할 것입니다. **PR에 대해 유지 관리자를 반복적으로 태그하지 마세요.**
