이 문서는 in-memory server(`langgraph dev`)와 local Docker server(`langgraph up`) 모두에 대해 LangGraph CLI 및 LangGraph Server에서 데이터가 처리되는 방식을 설명합니다. 또한 호스팅된 Studio frontend와 상호작용할 때 추적되는 데이터에 대해서도 설명합니다.

## CLI

LangGraph **CLI**는 LangGraph 애플리케이션을 빌드하고 실행하기 위한 command-line interface입니다. 자세한 내용은 [CLI 가이드](/langsmith/cli)를 참조하세요.

기본적으로 대부분의 CLI 명령 호출 시 단일 analytics event가 로깅됩니다. 이는 CLI 경험 개선의 우선순위를 더 잘 정하는 데 도움이 됩니다. 각 telemetry event에는 호출 프로세스의 OS, OS 버전, Python 버전, CLI 버전, 명령 이름(`dev`, `up`, `run` 등), 그리고 명령에 플래그가 전달되었는지를 나타내는 boolean 값이 포함됩니다. 전체 analytics 로직은 [여기](https://github.com/langchain-ai/langgraph/blob/main/libs/cli/langgraph-cli/analytics.py)에서 확인할 수 있습니다.

`LANGGRAPH_CLI_NO_ANALYTICS=1`을 설정하여 모든 CLI telemetry를 비활성화할 수 있습니다.

## LangGraph Server

[LangGraph Server](/langsmith/langgraph-server)는 애플리케이션 상태의 checkpoint, 장기 memory, thread metadata, assistant 및 유사한 리소스를 로컬 파일 시스템 또는 데이터베이스에 저장하는 것에 의존하는 지속 가능한 실행 runtime을 제공합니다. 저장 위치를 의도적으로 사용자 정의하지 않은 경우, 이 정보는 로컬 디스크(`langgraph dev`의 경우) 또는 PostgreSQL 데이터베이스(`langgraph up` 및 모든 배포의 경우)에 기록됩니다.

### LangSmith Tracing

LangGraph server를 실행할 때(in-memory 또는 Docker 모두), LangSmith tracing을 활성화하여 더 빠른 디버깅을 촉진하고 프로덕션 환경에서 graph state 및 LLM prompt의 관찰 가능성을 제공할 수 있습니다. server의 runtime 환경에서 `LANGSMITH_TRACING=false`를 설정하여 언제든지 tracing을 비활성화할 수 있습니다.

<a id="langgraph-dev"></a>
### In-memory development server

`langgraph dev`는 빠른 개발 및 테스트를 위해 설계된 단일 Python 프로세스로 [in-memory development server](/langsmith/local-server)를 실행합니다. 모든 checkpointing 및 memory 데이터를 현재 작업 디렉토리의 `.langgraph_api` 디렉토리 내 디스크에 저장합니다. [CLI](#cli) 섹션에서 설명한 telemetry 데이터를 제외하고, tracing을 활성화했거나 graph 코드가 명시적으로 외부 서비스에 연결하지 않는 한 데이터가 머신 외부로 나가지 않습니다.

<a id="langgraph-up"></a>
### Standalone Server

`langgraph up`은 로컬 패키지를 Docker 이미지로 빌드하고 API server, PostgreSQL container, Redis container의 세 가지 container로 구성된 [data plane](/langsmith/self-hosted)으로 server를 실행합니다. 모든 영구 데이터(checkpoint, assistant 등)는 PostgreSQL 데이터베이스에 저장됩니다. Redis는 event의 실시간 스트리밍을 위한 pubsub 연결로 사용됩니다. 유효한 `LANGGRAPH_AES_KEY` 환경 변수를 설정하여 데이터베이스에 저장하기 전에 모든 checkpoint를 암호화할 수 있습니다. 또한 `langgraph.json`에서 checkpoint 및 cross-thread memory에 대한 [TTL](/langsmith/configure-ttl)을 지정하여 데이터가 저장되는 기간을 제어할 수 있습니다. 모든 저장된 thread, memory 및 기타 데이터는 관련 API endpoint를 통해 삭제할 수 있습니다.

server가 유효한 라이선스를 가지고 있는지 확인하고 실행된 run 및 task의 수를 추적하기 위해 추가 API 호출이 이루어집니다. 주기적으로 API server는 제공된 license key(또는 API key)를 검증합니다.

[tracing](#langsmith-tracing)을 비활성화한 경우, graph 코드가 명시적으로 외부 서비스에 연결하지 않는 한 사용자 데이터가 외부에 저장되지 않습니다.

## Studio

[Studio](/langsmith/studio)는 LangGraph server와 상호작용하기 위한 그래픽 인터페이스입니다. 개인 데이터를 저장하지 않습니다(server로 보내는 데이터는 LangSmith로 전송되지 않습니다). Studio 인터페이스는 [smith.langchain.com](https://smith.langchain.com)에서 제공되지만, 브라우저에서 실행되며 로컬 LangGraph server에 직접 연결되므로 LangSmith로 데이터를 보낼 필요가 없습니다.

로그인한 경우, LangSmith는 디버깅 사용자 경험을 개선하는 데 도움이 되는 일부 사용 분석 데이터를 수집합니다. 여기에는 다음이 포함됩니다:

* 페이지 방문 및 탐색 패턴
* 사용자 작업(버튼 클릭)
* 브라우저 유형 및 버전
* 화면 해상도 및 viewport 크기

중요한 점은 애플리케이션 데이터나 코드(또는 기타 민감한 구성 세부 정보)는 수집되지 않는다는 것입니다. 이 모든 것은 LangGraph server의 persistence layer에 저장됩니다. Studio를 익명으로 사용하는 경우 계정 생성이 필요하지 않으며 사용 분석 데이터가 수집되지 않습니다.

## 빠른 참조

요약하면, CLI analytics를 끄고 tracing을 비활성화하여 server-side telemetry를 opt-out할 수 있습니다.

| Variable                       | 목적                      | 기본값                            |
| ------------------------------ | ------------------------- | -------------------------------- |
| `LANGGRAPH_CLI_NO_ANALYTICS=1` | CLI analytics 비활성화     | Analytics 활성화됨                |
| `LANGSMITH_API_KEY`            | LangSmith tracing 활성화   | Tracing 비활성화됨                |
| `LANGSMITH_TRACING=false`      | LangSmith tracing 비활성화 | 환경에 따라 다름                   |
