LangSmith를 시작하려면 계정을 생성해야 합니다. [여기](https://smith.langchain.com)에서 무료 계정에 가입할 수 있습니다. Google, GitHub 및 이메일을 통한 로그인을 지원합니다.

![Create account](/langsmith/images/create-account.png)

## API keys

LangSmith는 두 가지 유형의 API key를 지원합니다: Service Keys와 Personal Access Tokens입니다. 두 유형의 token 모두 LangSmith API에 대한 요청을 인증하는 데 사용할 수 있지만, 사용 사례가 다릅니다.

Service Keys와 Personal Access Tokens의 차이점에 대한 자세한 내용은 [admin concepts](/langsmith/administration-overview)를 참조하세요.

## API key 생성하기

LangSmith로 trace를 기록하고 evaluation을 실행하려면 요청을 인증하기 위한 API key를 생성해야 합니다. API key는 workspace 집합 또는 전체 organization으로 범위를 지정할 수 있습니다.

두 유형의 API key를 생성하려면 [Settings 페이지](https://smith.langchain.com/settings)로 이동한 다음 **API Keys** 섹션으로 스크롤하세요. service key의 경우 organization 범위 key와 workspace 범위 key 중에서 선택하세요. key가 workspace 범위인 경우 workspace를 지정해야 합니다. Enterprise 사용자는 key에 특정 role을 할당하여 권한을 조정할 수도 있습니다. key의 만료 기간을 설정하세요. key는 선택한 일수가 지나면 사용할 수 없게 되며, 선택한 경우 만료되지 않습니다. 그런 다음 **Create API Key**를 클릭하세요.

<Note>
API key는 한 번만 표시되므로 반드시 복사하여 안전한 곳에 보관하세요.
</Note>

![Create API key](/langsmith/images/create-api-key.png)

## SDK 구성하기

`LANGSMITH_API_KEY` 외에 다음 environment variable을 설정할 수 있습니다.

이는 EU instance를 사용하는 경우에만 필요합니다.

`LANGSMITH_ENDPOINT=https://eu.api.smith.langchain.com`

이는 둘 이상의 workspace로 범위가 지정된 key에만 필요합니다.

`LANGSMITH_WORKSPACE_ID=<Workspace ID>`
