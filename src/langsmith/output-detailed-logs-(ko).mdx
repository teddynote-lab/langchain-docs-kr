LangSmith package는 Python의 내장 메커니즘을 사용하여 동작에 대한 로그를 표준 출력으로 출력합니다.

## logging이 구성되어 있는지 확인

<Note>
기본적으로 Jupyter notebook은 로그를 표준 출력 대신 표준 에러로 전송하므로, 아래와 같이 logging을 구성하지 않으면 notebook cell 출력에 로그가 표시되지 않습니다.
</Note>

현재 Python 환경에서 logging이 표준 출력으로 로그를 전송하도록 구성되어 있지 않다면, 다음과 같이 명시적으로 활성화해야 합니다:

```python
import logging
# Note: this will affect _all_ packages that use python's built-in logging mechanism,
#       so may increase your log volume. Pick the right log level for your use case.
logging.basicConfig(level=logging.WARNING)
```

## logger의 verbosity 증가

문제를 디버깅할 때는 로그를 더 높은 수준의 verbosity로 증가시켜 표준 출력에 더 많은 정보가 출력되도록 하는 것이 유용합니다. Python logger는 기본적으로 `WARNING` log level을 사용하지만, 다른 값을 선택하여 다양한 수준의 verbosity를 얻을 수 있습니다. 값은 가장 낮은 verbosity부터 높은 순서로 `ERROR`, `WARNING`, `INFO`, `DEBUG`입니다. 다음과 같이 설정할 수 있습니다:


```python
import langsmith
import logging

# Loggers are hierarchical, so setting the log level on "langsmith" will
# set it on all modules inside the "langsmith" package
langsmith_logger = logging.getLogger("langsmith")
langsmith_logger.setLevel(level=logging.DEBUG)
```
