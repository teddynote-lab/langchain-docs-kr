**LangGraph Server**는 에이전트 기반 애플리케이션을 생성하고 관리하기 위한 API를 제공합니다. [assistants](/langsmith/assistants) 개념을 기반으로 구축되었으며, 이는 특정 작업을 위해 구성된 에이전트입니다. 또한 내장된 [persistence](/oss/langgraph/persistence#memory-store)와 **task queue**를 포함합니다. 이 다목적 API는 백그라운드 처리부터 실시간 상호작용까지 광범위한 에이전트 애플리케이션 사용 사례를 지원합니다.

LangGraph Server를 사용하여 [assistants](/langsmith/assistants), [threads](/oss/langgraph/persistence#threads), [runs](/langsmith/assistants#execution), [cron jobs](/langsmith/cron-jobs), [webhooks](/langsmith/use-webhooks) 등을 생성하고 관리할 수 있습니다.

<Tip>
**API reference**<br></br>
API endpoint와 데이터 모델에 대한 자세한 정보는 [API reference docs](https://langchain-ai.github.io/langgraph/cloud/reference/api/api_ref.html)를 참조하세요.
</Tip>

LangGraph Server의 `Enterprise` 버전을 사용하려면 Docker 이미지를 실행할 때 지정해야 하는 라이선스 키를 획득해야 합니다. 라이선스 키를 획득하려면 [영업팀에 문의](https://www.langchain.com/contact-sales)하세요.

다음 LangSmith [hosting](/langsmith/hosting) 옵션에서 LangGraph Server의 `Enterprise` 버전을 실행할 수 있습니다:

- [Cloud](/langsmith/cloud)
- [Hybrid](/langsmith/hybrid)

## Application structure

LangGraph Server 애플리케이션을 배포하려면 배포하려는 graph(s)와 의존성 및 환경 변수와 같은 관련 구성 설정을 지정해야 합니다.

배포를 위해 LangGraph 애플리케이션을 구조화하는 방법을 알아보려면 [application structure](/langsmith/application-structure) 가이드를 읽어보세요.

## Parts of a deployment

LangGraph Server를 배포할 때, 하나 이상의 [graphs](#graphs), [persistence](/oss/langgraph/persistence)를 위한 데이터베이스, 그리고 task queue를 배포하게 됩니다.

### Graphs

LangGraph Server로 graph를 배포할 때, [Assistant](/langsmith/assistants)를 위한 "blueprint"를 배포하는 것입니다.

[Assistant](/langsmith/assistants)는 특정 구성 설정과 결합된 graph입니다. graph당 여러 assistant를 생성할 수 있으며, 각각 동일한 graph로 제공될 수 있는 다양한 사용 사례를 수용하기 위해 고유한 설정을 가질 수 있습니다.

배포 시 LangGraph Server는 graph의 기본 구성 설정을 사용하여 각 graph에 대한 기본 assistant를 자동으로 생성합니다.

<Note>
우리는 종종 graph를 [agent](/oss/langgraph/workflows-agents)를 구현하는 것으로 생각하지만, graph가 반드시 agent를 구현할 필요는 없습니다. 예를 들어, graph는 애플리케이션 제어 흐름에 영향을 줄 수 있는 능력 없이 단순히 양방향 대화만 지원하는 간단한 챗봇을 구현할 수 있습니다. 실제로 애플리케이션이 더 복잡해지면, graph는 종종 [multiple agents](/oss/langchain/multi-agent)가 함께 작동하는 더 복잡한 흐름을 구현하게 됩니다.
</Note>

### Persistence and task queue

LangGraph Server는 [persistence](/oss/langgraph/persistence)를 위한 데이터베이스와 task queue를 활용합니다.

[PostgreSQL](https://www.postgresql.org/)은 LangGraph Server의 데이터베이스로 지원되며, [Redis](https://redis.io/)는 task queue로 지원됩니다.

[LangSmith cloud](/langsmith/cloud)를 사용하여 배포하는 경우, 이러한 구성 요소는 자동으로 관리됩니다. [자체 인프라](/langsmith/self-hosted)에 LangGraph Server를 배포하는 경우, 이러한 구성 요소를 직접 설정하고 관리해야 합니다.

이러한 구성 요소가 어떻게 설정되고 관리되는지에 대한 자세한 내용은 [hosting options](/langsmith/hosting) 가이드를 검토하세요.
