# 자체 Agent 구축하기

우리는 custom agent를 염두에 두고 Open Agent Platform을 구축했습니다. 몇 가지 사전 구축된 agent를 제공하지만, 자체 agent를 구축하고 OAP를 프로토타입 제작, 테스트 및 사용을 위한 플랫폼으로 활용하시기를 권장합니다! 이 가이드는 Open Agent Platform의 모든 기능과 호환되는 agent를 구축하는 데 도움을 드립니다.

## Platform 요구사항

OAP는 LangSmith 위에 구축되었으며, 이는 OAP에서 사용하기 위해 구축하는 모든 agent가 LangSmith에 배포된 LangGraph agent여야 함을 의미합니다.

## Agent 유형

custom agent를 구축할 때 세 가지 주요 유형을 생성할 수 있습니다:

1. **Standard Agent**: 특정 작업을 처리하는 단일 목적 agent입니다.
2. **Tools Agent**: MCP protocol을 통해 외부 tool에 액세스할 수 있는 agent입니다.
3. **Supervisor Agent**: 여러 다른 agent를 조율하고 조정할 수 있는 agent입니다.

## 개발 흐름

custom agent를 생성하기 위한 일반적인 개발 흐름은 다음과 같습니다:

1. LangGraph를 사용하여 로컬에서 agent를 개발하고 테스트합니다
2. LangSmith에 agent를 배포합니다
3. 배포된 agent에 연결하도록 Open Agent Platform을 구성합니다
4. OAP interface를 통해 agent를 테스트하고 개선합니다

## 시작하기

자체 agent 구축을 시작하려면 다음을 권장합니다:

1. [LangGraph framework](https://github.com/langchain-ai/langgraph)에 익숙해지기
2. 우리가 제공하는 예제 agent 학습하기:
   - [Tools Agent](https://github.com/langchain-ai/oap-langgraph-tools-agent)
   - [Supervisor Agent](https://github.com/langchain-ai/oap-agent-supervisor)
   - [Deep Research Agent](https://github.com/langchain-ai/oap-deep-researcher)
3. [Configuration](/labs/oap/custom-agents/configuration) 섹션에 자세히 설명된 대로 agent를 구성 가능하게 만드는 방법 이해하기

custom agent를 구축하면 Open Agent Platform interface를 통해 쉽게 관리하고 사용할 수 있는 특화된 AI assistant를 만들 수 있습니다.
