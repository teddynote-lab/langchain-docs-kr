# 모범 사례

Open SWE에서 최상의 결과를 얻기 위해 다음 가이드라인을 따르세요.

## 프롬프트 작성 팁

### 명확하고 직접적으로 작성하기

- 요청에 구체적인 파일 경로와 함수 이름을 포함하세요. Open SWE는 명확한 시작점이 주어졌을 때 가장 좋은 성능을 발휘합니다.
- 달성하고자 하는 것에 대한 구체적인 예시를 제공하세요. Open SWE가 무엇을 향해 작업하고 있는지 알 수 있도록 도달하고자 하는 최종 상태를 설명하세요.

### Custom Rule 생성하기

프로젝트별 컨텍스트를 제공하기 위해 리포지토리 루트에 `AGENTS.md` 파일을 생성하세요. 이는 Open SWE가 코드베이스의 규칙과 요구사항을 이해하는 데 도움이 됩니다.

<Tip>
  `AGENTS.md` 파일 설정에 대한 자세한 가이드는 [Custom Rules](/labs/swe/usage/custom-rules) 페이지를 참조하세요.
</Tip>

### 작업 범위를 적절하게 유지하기

- 요청당 하나의 특정 기능 또는 수정에 집중하세요
- 큰 변경사항을 더 작고 관리 가능한 작업으로 나누세요
- 단일 요청에 여러 관련 없는 변경사항을 결합하지 마세요

### 여러 작업 피하기

서로 다른 기능이나 수정에 대해서는 별도의 요청을 제출하세요. 이를 통해 Open SWE는 다음을 수행할 수 있습니다:

- 더 집중된 계획 생성
- 더 나은 오류 처리 제공
- 변경사항을 더 쉽게 검토 가능

## Model 선택

- **Claude Sonnet 4 (기본값)**: 계획 수립, 코드 작성 및 변경사항 검토를 위한 기본 모델입니다. 이 모델은 성능, 속도 및 비용의 최적 균형을 제공합니다.
- **Claude Opus 4.1**: 어렵거나 개방형 작업을 위한 더 크고 강력한 모델입니다. Opus 4.1은 더 비싸고 느리지만, 복잡한 작업에 대해 더 나은 결과를 제공합니다.

### 다른 Model 사용 피하기

Open SWE는 Anthropic, OpenAI 및 Google의 모든 모델을 선택할 수 있지만, 프롬프트는 Anthropic 모델에 특별히 최적화되어 있으며 다른 제공업체는 성능이 좋지 않습니다.

## Mode 선택

### `open-swe` vs `open-swe-max`

**`open-swe`**: Claude Sonnet 4 사용

- 대부분의 개발 작업에 적합
- 더 빠른 실행
- 비용 효율적

**`open-swe-max`**: Claude Opus 4 사용 (계획 수립 및 코드 작성 agent에만 해당)

- 고급 추론이 필요한 복잡한 작업용
- 어려운 문제에 대한 더 높은 품질의 출력
- 더 비싸지만 어려운 작업에 더 적합

### Auto vs Manual Label

**Auto Mode (`-auto` label)**

대부분의 작업에는 auto mode를 사용하는 것이 권장됩니다. Open SWE는 계획 수립에 매우 뛰어나며, 대부분의 경우 실행 전에 수동 검토가 필요하지 않습니다.

개방형이거나 매우 복잡한 작업에 대해 Open SWE를 실행하는 경우, 실행 전에 계획을 검토하기 위해 manual mode를 사용할 수 있습니다.

## Label 참조

- `open-swe`: Sonnet 4를 사용하는 Manual mode
- `open-swe-auto`: Sonnet 4를 사용하는 Auto mode
- `open-swe-max`: Opus 4.1을 사용하는 Manual mode
- `open-swe-max-auto`: Opus 4.1을 사용하는 Auto mode

<Note>
  개발 환경에서는 모든 label에 `-dev`를 추가하세요 (예: `open-swe-dev`, `open-swe-auto-dev`).
</Note>
