# Open SWE 사용하기

Open SWE는 코딩 에이전트와 상호작용할 수 있는 두 가지 주요 방법을 제공하며, 각각 다른 워크플로우와 사용 사례를 위해 설계되었습니다. 웹 인터페이스를 통한 직접적인 상호작용을 선호하든 GitHub webhook을 통한 자동화된 트리거를 선호하든, Open SWE는 여러분의 개발 프로세스에 맞춰 조정됩니다.

## 시작하기

### 데모 사용해보기

[swe.langchain.com](https://swe.langchain.com)에서 호스팅되는 데모를 통해 Open SWE의 기능을 탐색할 수 있습니다.

<Note>
  데모 애플리케이션은 자체 LLM API 키를 제공해야 합니다. 프로덕션 사용 또는 개발을 위해서는 [개발 설정 가이드](/labs/swe/setup/development)를 따라 자체 인스턴스를 설정하는 것을 권장합니다.
</Note>

### 워크플로우 선택하기

- **웹 인터페이스**: 대화형 개발, 실험, 그리고 에이전트의 계획 및 실행 프로세스를 직접 제어하고 싶을 때 가장 적합합니다
- **GitHub Webhooks**: 자동화된 워크플로우, 이슈 기반 개발, 그리고 GitHub 이슈를 통해 코딩 작업을 추적하는 팀 협업에 이상적입니다

## 다음 단계

<Tip>
  Open SWE의 핵심 기능을 이해하기 위해 [웹 인터페이스 가이드](/labs/swe/usage/ui)부터 시작한 다음, 개발 워크플로우에 자동화된 통합을 위해 [GitHub Webhooks](/labs/swe/usage/github)를 탐색하세요.
</Tip>

두 가지 사용 방법 모두 계획, 프로그래밍 및 관리를 위한 특수 그래프가 있는 동일한 기본 LangGraph 에이전트 아키텍처를 활용하여, Open SWE와 상호작용하는 방식에 관계없이 일관된 동작을 보장합니다.
