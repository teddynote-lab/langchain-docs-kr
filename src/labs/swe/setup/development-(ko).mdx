이 가이드는 로컬 개발을 위한 Open SWE 설정 과정을 안내합니다. 리포지토리를 클론하고, 의존성을 설치하고, 환경 변수를 구성하고, GitHub App을 생성하고, 개발 서버를 시작해야 합니다.

<Note>
  이 설정은 개발 목적입니다. 프로덕션 배포를 위해서는 URL을 조정하고 프로덕션용 별도의 GitHub App을 생성해야 합니다.
</Note>

## 사전 요구사항

시작하기 전에 다음이 설치되어 있는지 확인하세요:

- Node.js (버전 18 이상)
- Yarn (버전 3.5.1 이상)
- Git

## 설정 단계

<Steps>
  <Step title="리포지토리 클론">
    Open SWE 리포지토리를 로컬 머신에 클론합니다:

    ```bash
    git clone https://github.com/langchain-ai/open-swe.git
    ```
    ```bash
    cd open-swe
    ```

  </Step>

  <Step title="의존성 설치">
    리포지토리 루트에서 Yarn을 사용하여 모든 의존성을 설치합니다:

    ```bash
    yarn install
    ```

    이렇게 하면 monorepo workspace의 모든 패키지에 대한 의존성이 설치됩니다.

  </Step>

  <Step title="환경 파일 설정">
    환경 예제 파일을 복사하고 구성합니다:

    ```bash
    # Copy web app environment file
    cp apps/web/.env.example apps/web/.env
    ```
    ```bash
    # Copy agent environment file
    cp apps/open-swe/.env.example apps/open-swe/.env
    ```

    ### Web App 환경 변수 (`apps/web/.env`)

    다음 변수를 입력합니다 (GitHub App 값은 다음 단계에서 추가됩니다):

    ```bash Environment Variables [expandable]
    # API URLs for development
    NEXT_PUBLIC_API_URL="http://localhost:3000/api"
    LANGGRAPH_API_URL="http://localhost:2024"

    # Encryption key for secrets (generate with: openssl rand -hex 32)
    SECRETS_ENCRYPTION_KEY=""

    # GitHub App OAuth settings (will be filled after creating GitHub App)
    NEXT_PUBLIC_GITHUB_APP_CLIENT_ID=""
    GITHUB_APP_CLIENT_SECRET=""
    GITHUB_APP_REDIRECT_URI="http://localhost:3000/api/auth/github/callback"

    # GitHub App details (will be filled after creating GitHub App)
    GITHUB_APP_NAME="open-swe-dev" # this must match the name of your GitHub app, excluding spaces
    GITHUB_APP_ID=""
    GITHUB_APP_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----
    ...add your private key here...
    -----END RSA PRIVATE KEY-----
    "

    # List of GitHub usernames that are allowed to use Open SWE without providing API keys
    # This is only used in production. In development every user is an "allowed user".
    # Must be a valid JSON array of strings.
    NEXT_PUBLIC_ALLOWED_USERS_LIST='["your-github-username", "teammate-username"]'
    ```

    ### Agent 환경 변수 (`apps/open-swe/.env`)

    agent 환경 변수를 구성합니다:

    ```bash Environment Variables [expandable]
    # LangSmith tracing & LangSmith deployment
    LANGCHAIN_PROJECT="default"
    LANGCHAIN_API_KEY="lsv2_pt_..."  # Get from LangSmith
    LANGCHAIN_TRACING_V2="true"
    LANGCHAIN_TEST_TRACKING="false"

    # LLM Provider Keys (at least one required)
    ANTHROPIC_API_KEY=""  # Recommended - default provider
    OPENAI_API_KEY=""     # Optional
    GOOGLE_API_KEY=""     # Optional

    # Infrastructure
    DAYTONA_API_KEY=""    # Required. For cloud sandboxes

    # Tools
    FIRECRAWL_API_KEY=""  # For URL content extraction

    # GitHub App settings (same as web app)
    GITHUB_APP_NAME="open-swe-dev" # this must match the name of your GitHub app, excluding spaces
    GITHUB_APP_ID=""
    GITHUB_APP_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----
    ...add your private key here...
    -----END RSA PRIVATE KEY-----
    "
    GITHUB_WEBHOOK_SECRET=""  # Will be generated in next step

    # Server configuration
    PORT="2024"
    OPEN_SWE_APP_URL="http://localhost:3000"
    SECRETS_ENCRYPTION_KEY=""  # Must match web app value

    # CI/CD. See the /labs/swe/setup/ci for more information
    SKIP_CI_UNTIL_LAST_COMMIT="true"

    # List of GitHub usernames that are allowed to use Open SWE without providing API keys
    # This is only used in production. In development every user is an "allowed user".
    # Must be a valid JSON array of strings.
    NEXT_PUBLIC_ALLOWED_USERS_LIST='["your-github-username", "teammate-username"]'
    ```

    `NEXT_PUBLIC_ALLOWED_USERS_LIST` 환경 변수를 설정하지 않으면, 모든 사용자가 agent를 사용하기 위해 자신의 LLM API 키를 설정해야 합니다. 또한 `NEXT_PUBLIC_ALLOWED_USERS_LIST` 환경 변수(web 및 agent 배포 모두)에 액세스를 허용하려는 GitHub 사용자의 사용자 이름을 포함하지 않으면 webhook 기능(GitHub issue에 레이블을 추가하여 새 실행 트리거, PR 리뷰에서 agent 태그 지정 등)이 작동하지 않습니다.

    <Tip>
    `SECRETS_ENCRYPTION_KEY`는 다음 명령으로 생성하세요: `openssl rand -hex 32`. 이 키는 두 환경 파일에서 동일해야 합니다.
    </Tip>

  </Step>

  <Step title="GitHub App 생성">
    <Note>
    **GitHub App**(GitHub OAuth App이 아님)을 생성해야 합니다. 이들은 서로 다른 기능을 가진 다른 유형의 애플리케이션입니다. 개발 및 프로덕션 환경에 대해 별도의 GitHub app을 생성하는 것을 고려하세요.
    </Note>

    ### GitHub App 생성

    1. [GitHub App 생성 페이지](https://github.com/settings/apps/new)로 이동합니다
    2. 기본 정보를 입력합니다:
       - **GitHub App name**: 원하는 이름
       - **Description**: Open SWE 코딩 agent의 개발 인스턴스
       - **Homepage URL**: 리포지토리 URL
       - **Callback URL**: `http://localhost:3000/api/auth/github/callback`

    ### OAuth 설정 구성

    - ✅ **Request user authorization (OAuth) during installation** - 사용자가 web app에 로그인할 수 있도록 허용
    - ✅ **Redirect on update** - 권한 업데이트 후 사용자를 앱으로 리디렉션
    - ❌ **Expire user authorization tokens** - 토큰이 만료되지 않도록 유지

    ### Webhook 설정

    1. ✅ **Enable webhook**
    2. **Webhook URL**: ngrok과 같은 도구를 사용하여 로컬 서버를 노출해야 합니다:
       ```bash
       # Install ngrok if you haven't already
       # Then expose your local LangGraph server
       ngrok http 2024
       ```
       ngrok URL + `/webhooks/github`를 사용합니다 (예: `https://abc123.ngrok.io/webhooks/github`)

    3. **Webhook secret**: 이 값을 생성하고 저장합니다:
       ```bash
       openssl rand -hex 32
       ```
       이 값을 `apps/open-swe/.env`의 `GITHUB_WEBHOOK_SECRET`에 추가합니다

    ### 권한 구성

    **Repository permissions:**
    - **Contents**: Read & Write
    - **Issues**: Read & Write
    - **Pull requests**: Read & Write
    - **Metadata**: Read only (자동으로 활성화됨)

    **Organization permissions:** None

    **Account permissions:** None

    ### 이벤트 구독

    - ✅ **Issues** - webhook 기능에 필요
    - ✅ **Pull request review** - PR 태그 지정 기능에 필요
    - ✅ **Pull request review comment** - PR 태그 지정 기능에 필요
    - ✅ **Issue comment** - PR 태그 지정 기능에 필요

    ### 설치 설정

    - **Where can this GitHub App be installed?**:
      - 더 광범위한 테스트를 위해 "Any account" 선택
      - 또는 리포지토리로 제한하려면 "Only on this account" 선택

    ### App 생성 완료

    **Create GitHub App**을 클릭하여 설정을 완료합니다.

    ### App 자격 증명 수집

    앱을 생성한 후 다음 값을 수집하여 두 환경 파일에 추가합니다:

    - **GITHUB_APP_NAME**: 선택한 이름
    - **GITHUB_APP_ID**: "About" 섹션에서 찾을 수 있음 (예: `12345678`)
    - **NEXT_PUBLIC_GITHUB_APP_CLIENT_ID**: "About" 섹션에서 찾을 수 있음
    - **GITHUB_APP_CLIENT_SECRET**:
      1. "Client secrets" 섹션으로 스크롤
      2. "Generate new client secret" 클릭
      3. 생성된 값 복사
    - **GITHUB_APP_PRIVATE_KEY**:
      1. "Private keys" 섹션으로 스크롤
      2. "Generate a private key" 클릭
      3. `.pem` 파일을 다운로드하고 내용 복사
      4. 줄 바꿈에 `\\n`을 사용하여 한 줄로 포맷하거나 예제에 표시된 여러 줄 형식 사용
    - **GITHUB_APP_REDIRECT_URI**: 로컬 개발의 경우 `http://localhost:3000/api/auth/github/callback`, 프로덕션의 경우 `https://your-production-url.com/api/auth/github/callback`이어야 합니다.
    - **GITHUB_WEBHOOK_SECRET**: 이 값을 생성하고 저장합니다:
      ```bash
      openssl rand -hex 32
      ```
      이 값을 `apps/open-swe/.env` 및 `apps/web/.env`의 `GITHUB_WEBHOOK_SECRET`에 추가합니다.

    <Tip>
    GitHub App 자격 증명을 안전하게 보관하고 버전 관리에 커밋하지 마세요. `.env` 파일은 이미 `.gitignore`에 포함되어 있습니다.
    </Tip>

  </Step>

  <Step title="개발 서버 시작">
    모든 환경 변수가 구성되면 두 개발 서버를 시작합니다:

    **터미널 1 - LangGraph Agent 시작:**
    ```bash
    # apps/open-swe
    yarn dev
    ```
    이렇게 하면 `http://localhost:2024`에서 LangGraph 서버가 시작됩니다

    **터미널 2 - Web Application 시작:**
    ```bash
    # apps/web
    yarn dev
    ```
    이렇게 하면 `http://localhost:3000`에서 Next.js web app이 시작됩니다

    <Note>
    전체 기능을 위해서는 두 서버가 동시에 실행되어야 합니다. web app은 API 호출을 통해 LangGraph agent와 통신합니다.
    </Note>

  </Step>
</Steps>

## 확인

두 서버가 모두 실행되면:

1. **web app 방문**: `http://localhost:3000`으로 이동합니다
2. **GitHub 인증 테스트**: GitHub 계정으로 로그인을 시도합니다

<Tip>
  문제가 발생하면 두 터미널 창의 콘솔 로그에서 오류 메시지를 확인하세요. 일반적인 문제로는 누락된 환경 변수 또는 잘못된 GitHub App 구성이 있습니다.
</Tip>
